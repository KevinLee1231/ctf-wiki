# week3Maze

## 题目简述

程序把 16×24 地图存为一维数组，要求先输入起点坐标，再用 `w/a/s/d` 沿唯一通路从 `S` 走到 `E`。成功后 flag 格式为“最终行坐标-完整路径-最终列坐标”。

## 解题过程

索引计算为 `maze[24 * x + y]`，源码确认起点是 `(2, 4)`，终点是 `(5, 4)`。移动规则为 `w: x-1`、`s: x+1`、`a: y-1`、`d: y+1`。地图如下：

```text
************************
**********..........****
****S......********.****
*******************.****
*******************.****
****E***********....****
****.****........*******
****.****.**************
****.****.**************
****.****........*******
****.***********.*******
****.***********.*******
****.***********...*****
****.*************.*****
****...............*****
************************
```

沿 `.` 的唯一通路得到长度恰好为 71 的路径：

```text
ddddddwdddddddddssssaaasaaaaaaasssdddddddsssddssaaaaaaaaaaaaaawwwwwwwww
```

输入顺序为：

```text
2 4
ddddddwdddddddddssssaaasaaaaaaasssdddddddsssddssaaaaaaaaaaaaaawwwwwwwww
```

到达 `E` 时当前坐标为 `(5, 4)`，程序按 `0xGame{%d-%s-%d}` 输出：

```text
0xGame{5-ddddddwdddddddddssssaaasaaaaaaasssdddddddsssddssaaaaaaaaaaaaaawwwwwwwww-4}
```

## 方法总结

本题要同时还原地图索引、坐标方向和最终格式。路径本身不是完整 flag；程序使用到达终点后的 `x`、`y`，并把它们分别放在路径两侧。源码还限制路径长度不得超过 71，本题唯一正确路径正好达到上限。
