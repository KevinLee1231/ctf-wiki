# 指令神使

## 题目简述

附件是一个 Windows x64 程序。程序先校验输入长度和 `0xGame{...}` 格式，再变换其中的字符并与内置字符串比较，需要识别该字符变换并逆运算。

## 解题过程

在 `main` 中可以看到，输入总长度必须为 44，`sub_140001154` 负责检查 `0xGame{` 前缀及末尾 `}`，随后将待校验部分交给 `sub_140001118` 处理。程序最终比较的内置字符串为：

```text
2561p620-9462-6353-rq59-03340n99o7po}
```

字符变换只处理小写字母，核心表达式为：

```c
(ch - 84) % 26 + 'a'
```

在 `ch` 为小写字母时，这等价于字母循环位移 13 位，也就是 ROT13；数字、连字符和右花括号保持不变。ROT13 的加密与解密是同一个操作，因此可直接还原：

```python
import codecs

encoded = "2561p620-9462-6353-rq59-03340n99o7po}"
print("0xGame{" + codecs.decode(encoded, "rot_13"))
```

结果为：

```text
0xGame{2561c620-9462-6353-ed59-03340a99b7cb}
```

## 方法总结

识别字符变换时应把反编译表达式化简到字母索引空间。这里的 `% 26` 和固定偏移共同构成 ROT13，而 ROT13 是自逆变换；恢复内置比较串后补回程序单独校验的 `0xGame{` 前缀即可。
