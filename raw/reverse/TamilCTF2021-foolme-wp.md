# foolme

## 题目简述

题目给出一个 64 位 ELF，程序表面要求输入 passcode，成功后按 `TamilCTF{%s}` 输出原始输入。实际 `main` 的第一步调用会直接 `sleep`、打印提示并 `exit(0)`，程序无法按正常路径运行。

真正逻辑只能静态恢复：输入 26 字符，经过三层变换后与字符串 `TamilCTF{StRiNgs_C0mP4re5}` 比较。该字符串是中间校验目标，不是真 flag。

## 解题过程

### 关键观察

核心流程为：

```text
input[26]
  -> Stage 1: 字符算术/位运算，并乱序写入 26 元素数组
  -> Stage 2: 数组反转
  -> Stage 3: 再次按位置变换并生成字符串
  -> strcmp("TamilCTF{StRiNgs_C0mP4re5}")
```

每个输入字符只影响唯一输出位置，所以可以逐位枚举可打印字符，正向模拟三层流水线并匹配目标。

部分 Stage 1 操作会丢失信息，例如右移或整数除法，因此多个候选字符可映射到同一目标。最终候选需要结合语义短语选择。

### 求解步骤

写正向模拟器，对每个位置枚举 `0x20..0x7e`，保留能让目标对应字节匹配的候选。碰撞位置会出现多组可能，例如：

```text
input[0]  -> ['1', 'q']
input[2]  -> ['^', '_']
input[10] -> 16 个候选
input[21] -> ['H', 'I', 'J', 'K']
```

候选组合中有意义的 leetspeak 短语是：

```text
15_tH15_eZY_r3vErsE_cHalL?
```

程序成功时会把原始输入包进 `TamilCTF{}`，因此 flag 为：

```text
TamilCTF{15_tH15_eZY_r3vErsE_cHalL?}
```

## 方法总结

- 核心技巧：程序不可动态运行时，直接静态还原数据流并写正向模拟器。
- 识别信号：`strings` 中出现看似 flag 的字符串，但程序用它做 `strcmp` 目标时，要判断它是否只是中间值。
- 复用要点：有损变换会造成碰撞；先列出所有候选，再用题目语义、可读性或额外约束筛最终输入，不要强行选第一个结果。
