# archgame

## 题目简述

题目附件包含主程序 `ArchGam` 和 `challs/chall0.bin` 到 `chall49.bin` 共 50 个小二进制。主程序用 Unicorn 加载不同架构的 `chall*.bin`，每个关卡文件会根据输入返回不同常量，返回值既影响下一关选择，也会与累计 key 异或后用于解密后续文件。目标是恢复这些关卡之间的状态转移图，找到能走到目标返回值 `0xb7620858` 的路径。

## 解题过程

题目主体部分由 Unicorn 实现，并附加了数个架构的二进制文件。

程序主要流程：

1. 创建 Unicorn 并加载对应的架构文件，起始文件为 chall14
2. 加载时使用积累Key对bin文件解密
3. 每一个bin文件根据用户的不同输入，输出不同的返回值，一共有 7 种可能
4. 每一个 bin 执行完成后，将返回值与积累key异或并更新到积累Key
5. 用上一个 bin 的返回值查找下一个 bin 程序
6. 最终要使程序执行到返回值为0xb7620858的bin程序


解题思路：
解题关键是分析每一个 bin 文件的 7 种返回值，找到一条路径到 0xb7620858 返回值。

## 方法总结

- 核心技巧：把多架构 Unicorn 执行链转化为图搜索问题，节点是 `chall*.bin` 和架构模式，边是每个关卡可能返回的常量。
- 识别信号：附件同时出现大量小二进制和一个主调度程序，且运行流程会按返回值选择下一文件时，应先恢复调度逻辑、解密 key 更新方式和目标终止条件。
- 复用要点：不要逐个手工逆所有关卡。先用 Capstone 按架构批量反汇编，抽取返回常量，结合 key 异或规则递归展开路径；遇到大小端或架构模式字段时要跟随主程序的元数据，而不是固定一种反汇编模式。
