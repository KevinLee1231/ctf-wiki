# Counter-Strike Global Offensive Aimbot

## 题目简述

题目伪装成 CS:GO 辅助程序。它读取游戏进程数据，并在按下 Delete 时让准星依次射击一组墙面坐标。这些坐标并非用于瞄准敌人，而是编码了一段由弹孔画出的文字。

## 解题过程

不必安装或运行 CS:GO。静态分析主循环，可以找到一个很大的 `Vector3` 常量数组；每个元素由三个单精度浮点数组成：

```cpp
class Vector3 {
public:
    float x, y, z;
};
```

程序选择玩家头部前方、垂直于 $x$ 轴的墙面，因此数组中的 $x$ 基本为零。$y$ 控制左右移动，$z$ 控制上下移动。把二进制中的连续浮点数据按每组三个值恢复，再只取 $(y,z)$ 作为二维笔画位移。

数组用较大的横向移动分隔字符，其余常见步长为 `40`、`50`、`60` 或 `70`。从原点开始逐项累加位移并连接相邻点，可以依次画出：

```text
V A C B A N
```

源码中 `WorldToScreen` 会把这些墙面位置换算成屏幕坐标并调用鼠标点击；静态绘图等价于在游戏里观察弹孔，但不受游戏版本、窗口标题和内存偏移变化影响。

题目接受短答案或完整格式：

```text
VACBAN
UMDCTF{VACBAN}
```

## 方法总结

- 核心技巧：识别大量三浮点常量不是普通配置，而是二维笔画数据。
- 固定为零的 $x$ 说明所有点位于同一平面，真正的信息在 $y$、$z$ 两个分量。
- 游戏内存偏移会随版本变化；离线提取常量数组比运行辅助程序更安全、稳定。
- 对坐标序列应检查它表示绝对点还是相对位移；本题重复步长和字符分隔位移明显指向“海龟绘图”式累加。
