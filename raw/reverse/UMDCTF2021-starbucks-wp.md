# Starbucks

## 题目简述

题目给出 Java 类文件，多个名称无意义的函数互相调用。flag 被拆在字符串变换链中，主流程实际执行 `f3()`，而 `f3()` 组合了 `f2()` 与 `f1()`。

## 解题过程

使用 `javap` 或 Java 反编译器查看所有私有方法：

```bash
javap -c -p Starbucks.class
```

控制流表明：

```text
f3() -> f1(f2(...))
```

分别记录 `f2` 对输入的字符重排/替换，再把结果送入 `f1`，不要把两个函数当作互不相关的诱饵。最稳妥的方法是把反编译后的纯函数复制到一个最小 Java/Python 验证脚本，按照实际调用顺序运行。

最终输出：

```text
UMDCTF-{pyth0n_1s_b3tt3r}
```

## 方法总结

面对大量短函数时，先从入口建立调用图，再只追踪能到达比较或输出点的路径。函数名不能说明真实语义，组合顺序也不能交换；动态调用原类或重写等价纯函数，都应以反编译字节码的参数顺序为准。
