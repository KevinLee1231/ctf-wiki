# week2_re1

## 题目简述

题目只给出一段 x86-64 汇编文本和加密数组 `enc[]`，不需要运行二进制。汇编逻辑把数组原地从后向前变换，最终打印结果。直接按汇编语义正向模拟即可得到 flag。

## 解题过程

核心函数先计算字符串长度，再令 `i = len - 2` 从后向前遍历：

```text
diff = input[i] - input[i + 1]
input[i] = (diff + i) & 0xff
```

注意 `input[i + 1]` 是已经被修改后的值，因为循环是倒序执行。

脚本：

```python
enc = [
    210,204,198,223,171,104,149,192,148,103,103,94,82,86,
    140,136,136,129,79,84,83,80,72,119,121,79,87,75,70,
    126,121,75,73,115,122,78,77,71,65,67,140,125,
]

buf = enc[:]
for i in range(len(buf) - 2, -1, -1):
    buf[i] = (buf[i] - buf[i + 1] + i) & 0xff
print(bytes(buf).decode())
```

输出：

```text
flag{49be7981-6d3e-4341-a089-5f190d898437}
```

## 方法总结

- 核心技巧：按汇编原始顺序模拟原位变换。
- 识别信号：没有可执行文件但给出完整汇编和静态数组时，先判断是加密还是解密路径。
- 复用要点：原位数组题最容易错在遍历方向；倒序循环中后一个元素已经是新值。
