# UMDCTF 2025 - sudoku

## 题目简述

服务把输入的每个字符转换为整数列表 `s`，随后计算一条极长的 Python 链式比较。
只有前面的所有比较都成立，表达式末尾的 `print(open("flag.txt").read())` 才会被
求值。题目虽然名为 Sudoku，但实际是约束满足问题；没有更合适的安全方向，因此放入
`_unclassified`。

## 解题过程

程序的整体结构可以缩写为：

```python
(s := [*map(int, input("input: "))]) \
    is not s[69] == s[68] > s[58] == ... != s[23] \
    is not print(open("flag.txt").read())
```

Python 会把 `a < b < c` 解释为 `a < b and b < c`，并短路求值。这里同理：
若中间任意一个 `==`、`>` 或 `!=` 不成立，末尾 `print` 就不会执行；若全部成立，
最后才调用 `print`。所以目标不是寻找代码注入，而是给 81 个位置赋值，使整条约束链
为真。

官方 `solve.py` 建立 81 个 Z3 整数：

```python
from z3 import Int, Solver

s = [Int(f"a{i}") for i in range(81)]
solver = Solver()
for x in s:
    solver.add(x > 0, x < 10)
```

随后把原表达式按“左操作数、运算符、右操作数”拆成相邻三元组。例如：

```text
s[69] == s[68]
s[68] > s[58]
s[58] == s[26]
```

每个三元组作为独立约束加入求解器。约束可满足时，模型给出的 81 位输入是：

```text
224945891227236799775446158763132236157152174561681439614389755963829948537883684
```

把该字符串提交给服务，所有比较依次通过，求值到表达式末端并打印：

```text
UMDCTF{has_operator_chaining_gone_too_far}
```

## 方法总结

面对超长单行表达式，应先按语言语义恢复执行条件，而不是手工阅读每个索引。Python
链式比较共享中间操作数且会短路，恰好可以直接转成 SMT 约束。把输入域限制为
$1$ 到 $9$，再逐个加入二元关系，就能稳定得到一组可行解；本题并不要求满足传统
Sudoku 的行列宫规则。
