# dotNet(d0tN3t)

## 题目简述

附件的核心逻辑位于 .NET DLL，用 dnSpy、ILSpy 等工具即可还原 C#。程序逐字符执行加法、异或和索引平方异或，最后强制转换为 `byte` 与密文数组比较。

## 解题过程

反编译后的关键表达式是：

```csharp
(byte)(((byte)text[i] + 0x72 ^ 0x72) ^ i * i)
```

C# 中加法优先于按位异或，且最终 `(byte)` 会把结果截断到低八位。因此正向关系为：

$$
c_i = (((p_i + 114) \oplus 114) \oplus i^2) \bmod 256.
$$

异或自反，逆向时先撤销 $i^2$ 和 114，再减 114，并显式 `& 0xff` 模拟 C# 的字节截断：

```python
encrypted = [
    173, 146, 161, 174, 132, 179, 187, 234, 231, 244, 177,
    161, 65, 13, 18, 12, 166, 247, 229, 207, 125, 109, 67,
    180, 230, 156, 125, 127, 182, 236, 105, 21, 215, 148,
    92, 18, 199, 137, 124, 38, 228, 55, 62, 164,
]

plain = bytes(
    ((value ^ (index * index) ^ 114) - 114) & 0xFF
    for index, value in enumerate(encrypted)
)
print(plain.decode())
```

输出：

```text
moectf{7ce581d2-b2ab-4ceb-9bbe-435873083db6}
```

## 方法总结

.NET 题通常先用托管代码反编译器恢复高层语义，再重点检查显式类型转换。Python 整数不会像 C# `byte` 那样自动回绕，逆运算若遗漏 `& 0xff`，中间负数会导致错误字符。
