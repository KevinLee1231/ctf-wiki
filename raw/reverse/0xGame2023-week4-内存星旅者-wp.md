# 内存星旅者

## 题目简述

附件是一个 Windows x64 程序。正确参数会触发 flag 解密，程序将结果短暂写入临时目录后立即删除；与其竞争读取临时文件，更稳定的做法是静态恢复参数和内置密文。

## 解题过程

`main` 调用 `sub_140001510` 解析参数，成功后把计算出的整数传给 `sub_140001600`；失败分支只输出“无效的启动参数，星门关闭！”。

参数解析逻辑要求同时出现 `-0xGame` 和 `-Portal=`。继续跟踪校验分支，可得正确的 Portal 值以及传入解密函数的密钥均为十进制 `1897488`：

```powershell
./内存星旅者.exe -0xGame -Portal=1897488
```

`sub_140001600` 先用 `GetTempPathA` 获取临时目录并拼接文件名 `flag`，再遍历 9 个内置 DWORD。每个 DWORD 与密钥异或后恰好得到 4 个小端序 ASCII 字节，最后在两侧拼接 `0xGame{` 与 `}`。写出文件后，程序立即调用 `DeleteFileA` 删除它。

无需运行程序或阻止删除，直接复现静态解密即可：

```python
import struct

key = 1897488
encrypted = [
    0x6278C174,
    0x3225CC22,
    0x3728953D,
    0x632CD976,
    0x3031C328,
    0x2D2EC272,
    0x332EC325,
    0x3179CD73,
    0x397EC673,
]

body = b"".join(struct.pack("<I", value ^ key) for value in encrypted).decode()
print(f"0xGame{{{body}}}")
```

输出为：

```text
0xGame{d5db2892-a47f-0c87-0b62-5723c9e1c2b9}
```

## 方法总结

临时文件只是题目制造的动态干扰，并不是必须利用的取证窗口。只要从参数校验中恢复 XOR 密钥，再按程序实际采用的 32 位分组和小端字节序复现解密，就能确定性得到 flag，避免文件刚创建就被删除造成的竞态。
