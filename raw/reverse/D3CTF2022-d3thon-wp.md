# d3thon

## 题目简述

这道题目从设计到完成的整个过程比较匆忙，题目也本身有很多设计的不合理的地方，导致虚拟机内部信息泄露过于严重，大大降低了本题的难度，希望师傅们谅解。

题目的核心部分是一个 Python3.10.2 编写的简易虚拟机，并进行了简单的变量名称混淆，最后使用Cython0.29.28 编译出 so。由于没有对 so 的 import 做任何限制，并且 bcode.lbc 中的代码一定会被逐句解释，于是一种可能的解法就是将库 import 进去，逐条语句测试功能，可以非常快速地测出每一条语句的作用：

## 解题过程

`byte_analyzer` 模块暴露的接口很少，核心只有 `analize(string)`，模块内还有 `Functions`、`Variables`、`test` 等全局对象。运行题目给出的测试字符串时可以观察到它按自定义字节码解释输入。

## 方法总结

- 核心技巧：题目没有限制 so 的 import，且 `bcode.lbc` 会被逐句解释，因此动态导入库并逐条测试 bytecode 语义是最快路线。
- 静态路线：如果走 Cython so 逆向，先从全局字符串表定位 `_pyx_pymod_exec_byte_analizer`，再围绕 `import re`、`Variables = {}` 等高语义字符串恢复解释器主逻辑。
- 复用要点：Cython 生成代码会有大量版本检查、异常处理和边界分支，分析时应先筛掉模板噪声，把精力放在字符串表、全局对象初始化和真正的分派逻辑上。
