# 虚拟化神

## 题目简述

程序的授权状态跨进程保存在同目录的 `config.txt` 中。输入注册码后并不会立即解锁，而是写入一个布尔状态，下一次启动时再读取该状态决定是否显示 flag；因此可以直接篡改持久化配置绕过注册码校验。

## 解题过程

先运行程序并随意输入错误注册码，退出前观察到同目录生成 `config.txt`，其中记录的失败状态为 `0`（false）。将其改为 `1`（true）并保存，再次启动程序，读取配置的分支便会认为验证已经通过并解锁 flag。

反编译逻辑可概括为：

```c
FILE *stream = fopen("config.txt", "r");
if (stream != NULL && fgets(buffer, 2, stream) != NULL && atoi(buffer) == 1) {
    print_decrypted_flag();
}
```

若不修改文件，也可以静态分析注册码比较和内置数据解密逻辑，或在解密完成后动态查看内存；但直接修改不受保护的状态文件是本题最短路径。

## 方法总结

客户端本地的明文布尔值不能作为可信授权依据，用户可以直接修改它。分析带“重启后生效”提示的程序时，应重点检查配置文件、注册表和其他持久化状态，并通过反编译确认状态值与受保护功能之间的控制流关系。
