# VideoPlayer

## 题目简述

题目是一个带 VMP 保护和反调试的 Windows 视频播放器。程序存在后门账户，但登录还会校验机器信息；登录后播放 `.mp0` 文件时，会用机器信息 MD5 派生的数据参与视频解密。目标是绕过反调试和机器校验，进入播放逻辑，拿到解密后的 mp4 并从视频中读出 flag。

附件 `VideoPlayer.zip` 中只有 `VideoPlayer.exe` 和 `video.mp0`。因此 `.mp0` 不是普通媒体文件，而是需要通过播放器内部解密逻辑恢复的加密视频；分析重点是让程序带着正确的机器信息 MD5 进入播放函数，并从返回的 `std::vector` 中导出解密后的 mp4 数据。

## 解题过程

### 反调试处理与 VMP 脱壳 Dump

#### 法1：TitanHide过反调试

这边反调试使用TitanHide驱动，Github可以搜索到相关项目，运行TitanHide.sys需要配置环境。

使用VKD工具的target64中的vminstall在虚拟机中运行安装，会多出来一个引导启动，重启电脑选择新的引导启动就可以，他会进入内核调试模式，禁止驱动强制签名以及关闭PG，也就可以让我们加载titanhide驱动。

运行titanhide.sys，将titanhide的dbg相关插件文件放入dbg的plugins文件夹中，运行dbg即可调试VMP程序。

#### 法2：CheatEngine Veh Debugger

运行软件，CE附加，调试选项选择Veh debugger，即可直接调试VMP保护的程序。

## 方法总结

- 核心技巧：用 TitanHide 或 CE VEH 绕过 VMP 反调试，断 `GetSystemTimeAsFileTime` 找 OEP 并 Dump；再动态绕过后门机器校验，复用目标机器信息 MD5 解密视频。
- 识别信号：后门用户名存在但登录失败，同时程序读取系统信息并 MD5，说明还有设备绑定校验；播放器打开自定义视频格式后没有真正播放逻辑，说明解密结果可能只在内存中。
- 复用要点：只改校验函数返回值不够，后续解密仍会使用 MD5 vector；需要把 MD5 数据本身改为目标值。解密函数返回 `std::vector` 时，可以用头尾指针相减得到大小，再用 Scylla dump memory 导出结果。
