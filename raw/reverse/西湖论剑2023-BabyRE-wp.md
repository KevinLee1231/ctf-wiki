# BabyRE

## 题目简述

题目给出一个 32 位 Windows PE，程序读取用户输入，成功时输出 `Good, flag is DASCTF{your input}`。输入必须是数字字符。核心校验分两层：先把输入按自定义 base8 编码后与 96 字节目标数字串比较，再在 `atexit` 回调中用输入尾部 6 字节作为 RC4 key，对 112 字节编码结果做 RC4 校验。

真正的 flag 不是完整输入 `DASCTF{...}`，而是程序要求输入的 48 位数字串，最后外包 `DASCTF{}`。

## 解题过程

### 关键观察

程序通过 Hook `GetLastError` 初始化数据，并注册 `atexit(FUN_00405770)`，因此最终校验不在主函数返回前直接完成。主要流程如下：

```text
读入 48 字节数字串
  -> 自定义 base8 编码
  -> 比较 encoded[16:112] 与目标数字串
  -> 退出时进入 atexit 回调
  -> RC4(key=input[42:48]) 加密 encoded[:112]
  -> 比较 112 字节预期密文
```

自定义 base8 每 3 个输入字节产生 8 个字符，编码表运行时由 `0xcf..0xc8 ^ 0xff` 还原为字符 `0..7`。输入结构为：

```text
input[0:6]    编码前缀，不参与第一层比较
input[6:42]   编码后对应 96 字节目标数字串
input[42:48]  同时作为 RC4 key
```

### 求解步骤

先暴力 6 位十进制 RC4 key，检查 `RC4_decrypt(expected, key)[16:]` 是否等于第一层目标数字串。找到 key 后，再把完整编码结果按 base8 解码回 48 字节输入。

求解结果：

```text
RC4 key: 807391
input : 561516915572239428449843076691286116796614807391
flag  : DASCTF{561516915572239428449843076691286116796614807391}
```

## 方法总结

- 核心技巧：识别退出回调中的第二层校验，不要只看主函数里的第一层 `memcmp`。
- 识别信号：`atexit`、Hook API、运行时初始化编码表、输入尾部被当作密钥使用时，要把校验链完整串起来。
- 复用要点：多层编码/加密题先确定数据流边界；本题先用 RC4 约束确定尾部 key，再反解 base8，比直接枚举 48 位数字稳定得多。
