# UMDCTF2022 Dragon Pit Writeup

## 题目简述

程序读取最多 20 个字符，与运行时构造的 `expected` 数组比较；输入正确后，又用输入中的一个字符和几次整数运算拼出 flag。源码和未去符号二进制都已给出，因此直接还原字符数组即可。

## 解题过程

初始期望值为：

```c
char expected[] = {
    'w','h','0','n','T','h','e','D','r','0',
    'g','s','H','P','i','s','0','0','0','0',0
};
```

其中若干位置会被覆盖。已知 `z = 2`、`a = 13`：

```c
expected[2]  = (z*z*a) - (z/z);  // 51  -> '3'
expected[9]  = z*z*a;            // 52  -> '4'
expected[16] = 49;               // '1'
expected[18] = z*z*a;            // '4'
expected[19] = z*z*a + z*z;      // 56  -> '8'
```

所以应输入恰好 20 个字符：

```text
wh3nTheDr4gsHPis1048
```

`fgets(response, 21, stdin)` 最多接收 20 个正文字符，因此换行不会进入数组，末尾会自动补 `\0`，可以通过 `strcmp`。

成功分支先放入 `UMDCTF{BluSt0XXXXXXX`，再修改后七个位置：

```c
flag[13] = 'l';
flag[14] = '3';
flag[15] = 'd';
flag[16] = 'r';
flag[17] = response[9];  // '4'
flag[18] = 'g';
flag[19] = '}';
```

最终得到：

```text
UMDCTF{BluSt0l3dr4g}
```

## 方法总结

这类简单校验题不需要暴力枚举或符号执行。先区分数组的初始值与后续覆盖，再把整数表达式按字符编码解释即可。还要注意 `fgets` 的长度包含终止符：这里 20 字节正文恰好填满有效输入，不会把换行带入比较。
