# But Like When

## 题目简述

题目给出一个 Go 可执行文件，内部嵌入近一万条外观可信的假 flag。程序根据当前日期计算索引，只有“时间正确”时才打印目标项；直接运行或简单搜字符串无法判断哪一条是真的。

## 解题过程

逆向 `main` 可还原日期逻辑：

```go
now := time.Now()
oneYearFromNow := now.AddDate(1, 0, 0)
dateStr := oneYearFromNow.Format("01/02/2006")
dateParts := strings.Split(dateStr, "/")
```

虽然源码注释写的是“各位数字求和”，实际代码把月、日、年三个完整字段分别转成整数，因此计算的是：

```text
sum = month + day + year
```

后面的关键条件是 `sum == 2031`，满足时打印 `flags[sum]`。最直接的静态解法是在反编译器中确认索引后，导出字符串切片第 $2031$ 项；也可以在调试器中把 `sum` 改为 `2031`。

若按比赛时的年份复现，程序会先把 2023 年的系统时间加一年，因此需要令
$2024+\text{month}+\text{day}=2031$。例如把时间伪造为 2023 年 4 月 3 日，加一年后得到 2024 年 4 月 3 日，三项之和正好为 $2031$。

目标索引处的字符串为：

```text
UMDCTF{dictionary_time_travel_extremely_crazy}
```

## 方法总结

- 核心技巧：沿最终打印参数反向追踪索引，而不是在近万条候选字符串中凭语义猜测。
- 源码注释与真实实现不一致；这里是日期字段求和，不是日期各位数字求和。
- 时间依赖程序可通过伪造系统时间、调试器改变量或静态读取目标索引复现，静态方法最不受当前年份影响。
