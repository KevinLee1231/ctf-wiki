# clothes

## 题目简述

附件是一个使用 ASPack 压缩壳保护的 32 位 Windows PE。壳只负责隐藏原始入口点和代码段；脱壳后的校验逻辑很短：程序读取最多 31 字节，并逐字节检查

$$
\text{input}[i]\oplus y[i]=x[i],
\qquad 0\le i<30.
$$

因此主障碍是识别并越过 ASPack 的解压存根，异或恢复只是脱壳后的验证步骤。

## 解题过程

### 1. 定位原始入口点

壳识别工具或入口点附近的 ASPack 特征可以确认文件被压缩。用 OllyDbg/x32dbg 调试时，可按常见的 ESP 定律定位解压结束位置：

1. 从壳入口运行到保存通用寄存器的 `PUSHAD`，记录执行后的 `ESP`；
2. 对该栈位置设置硬件访问断点并继续运行；
3. 解压存根恢复寄存器时会在 `POPAD` 附近触发断点；
4. 单步越过随后的跳转，落点即原始入口点 OEP；
5. 若要保存独立文件，再转储内存映像并重建导入表；仅做本题时，也可直接在当前进程中继续分析。

这几步就是题目外部提示中“手动脱壳”的必要信息，因此不再依赖原提示链接。

### 2. 恢复输入

到达 OEP 后，搜索欢迎字符串可以定位 `main`。源码中的关键循环为：

```c
for (int i = 0; i < 30; i++) {
    if ((input[i] ^ y[i]) != x[i]) {
        puts("Disconnected.");
        return 1;
    }
}
```

异或满足自反性，所以

$$
\text{input}[i]=x[i]\oplus y[i].
$$

```python
x = bytes([
    0x73, 0x45, 0x2b, 0x47, 0x57, 0x69, 0x53, 0x0d,
    0x44, 0x4c, 0x2e, 0x2f, 0x05, 0x6a, 0x13, 0x4d,
    0x57, 0x31, 0x4b, 0x22, 0x58, 0x06, 0x49, 0x71,
    0x4c, 0x6a, 0x32, 0x64, 0x18, 0x45,
])
y = bytes([
    0x1e, 0x2a, 0x4e, 0x24, 0x23, 0x0f, 0x28, 0x39,
    0x71, 0x3c, 0x4f, 0x4c, 0x6e, 0x35, 0x22, 0x3e,
    0x08, 0x02, 0x31, 0x7d, 0x2c, 0x36, 0x16, 0x04,
    0x22, 0x1a, 0x53, 0x07, 0x73, 0x38,
])

flag = bytes(left ^ right for left, right in zip(x, y))
print(flag)
```

输出为：

```text
moectf{45pack_1s_3z_t0_unpack}
```

## 方法总结

- 核心技巧：利用解压存根保存、恢复寄存器时对同一栈帧的访问，定位 ASPack 跳转到 OEP 的时刻。
- 识别信号：入口点位于壳存根、导入表很小、代码在运行后才展开时，应先脱壳再分析业务校验。
- 复用要点：手动脱壳必须在副本或调试进程中操作；落到 OEP 后再按普通逆向流程找字符串、交叉引用和校验循环。
