# DyDebug

## 题目简述

程序的静态解密逻辑较长，但校验流程很直接：运行时先用内置密文与密钥生成明文缓冲区，再逐字节把用户输入与该缓冲区比较。由于正确 flag 在比较前已经完整存在于进程内存中，可以通过动态调试直接观察，而无需手工重写全部解密算法。

## 解题过程

在 IDA 伪代码中定位输入缓冲区与 `decrypt_string()` 返回值逐字节比较的位置。关键数据流可收敛为下面的伪代码，其中 `v6` 是解密后的目标缓冲区，密钥字符串为 `BV1Rs411S77f`：

```c
v6 = decrypt_string(ciphertext_hex, "BV1Rs411S77f");
for (i = 0; i < 44; ++i) {
    if (input[i] != v6[i])
        exit(0);
}
```

在比较循环开始前下断点并启动调试。断下后跟随 `v6` 指针，或在反汇编层查看传入比较逻辑的目标地址，即可在内存中读到：

```text
0xGame{91f2c64e-057d-4191-8868-9a8c0847b2c0}
```

这个方法成立的前提是程序在本地完成明文解密后再比较。如果程序比较的是哈希或逐块流式结果，内存中未必会出现完整 flag。

## 方法总结

- 核心技巧：在“解密完成、比较开始”之间设置断点，直接读取运行时明文。
- 识别信号：静态解密函数复杂，但比较一侧是运行时生成的连续缓冲区。
- 复用要点：先追踪数据流并找明文生命周期，不要看到复杂算法就立即硬逆；动态调试应停在明文已生成且尚未销毁的位置。
