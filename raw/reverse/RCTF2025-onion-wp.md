# Onion

## 题目简述

附件由 Linux 启动器 `onion2` 和字节码 `full_vmcode` 组成。启动器先读入 50 个十进制无符号 64 位整数，再执行一台自定义 VM；VM 的每一层只校验其中一个整数，校验通过后才用该整数作为 RC4 密钥解开下一层代码。最终 flag 被包在 50 层依次展开的自修改代码中，题名 “Onion” 即来自这一结构。

## 解题过程

### 还原 VM 与内存布局

将启动器与仓库中的 Rust 源码对照，可以确认 VM 具有 64 KiB 字节内存、8 个 64 位寄存器、16 位 `PC`、两个基址寄存器 `IX/IY`、向低地址增长的栈和零标志。启动器的关键装载流程是：

- 把 50 个输入按小端 `u64` 依次写到 `0xE000`，共 400 字节；
- 把 26497 字节的 `full_vmcode` 装到低地址；
- 从 `PC=0` 开始解释执行。

指令集中除了跳转、装载、存储、位运算、比较和栈操作，还支持按编号登记与调用 VM 函数。初始化阶段注册了三个关键函数：

| 函数编号 | 作用 |
|---:|---|
| `0x01` | 输出 `Fail` 并终止 |
| `0x10` | 以寄存器中的 64 位整数按小端编码成 8 字节 RC4 密钥，就地解密一段 VM 内存 |
| `0x20` | 执行 27 轮 SPECK-64/128，加密一个 64 位数据块 |

VM 没有原生的 64 位加减法。生成器把加法展开为反复计算 `carry=(a&b)<<1; a^=b`，把减法展开为补码加法；因此反汇编中会看到很长的 `AND/XOR/SHL/JZ` 循环。识别这些固定宏后，应把整段折叠回 `+ constant` 或 `- constant`，而不是逐条分析。

### 单层校验的等价逻辑

每层虽使用不同常量和输入下标，但语义完全一致：

```text
x = inputs[index]
x = 按顺序执行若干次 64 位模 2^64 的 +、-、^
y = SPECK64_128_encrypt(x, level_key)
if y != target:
    FAIL()
else:
    RC4(key=inputs[index].to_bytes(8, "little"),
        data=vmcode[next_level:])
    jump next_level
```

这里的 SPECK 是可逆分组密码，不是哈希。字节码同时给出了 128 位密钥和比较目标，所以无需爆破，也不适合把整条路径交给符号执行器：先对 `target` 做 SPECK 解密，再逆序撤销线性算术即可直接求回该层输入。

总 WP 展示的第一层实际检查 `inputs[2]`。折叠宏后，SPECK 前的运算为：

```text
x = inputs[2]
x -= 0x48f0e6421ac66dea
x ^= 0x5074d85b9194e696
x -= 0x5566488c9c5cf234
x ^= 0x8cb331163a92fc19
```

SPECK 密钥的两个 64 位部分是 `0x36b1cc9fe433713d`、`0xf97646d69c84ebd8`，比较目标为 `0xda19ba6b81c83f61`。先解密目标得到 `0x38377082e137ecb9`，再按相反顺序执行 `^`、`+`、`^`、`+`，得到：

```text
inputs[2] = 0xa28f38bd0463522c
```

### 逐层剥离

后续层在初始文件中仍是密文，不能一次性扫描出全部常量。可靠做法是循环执行以下步骤：

1. 只反汇编当前已解开的层，按生成器的固定字节模式提取输入偏移、算术宏、SPECK 密钥、比较目标和下一层地址。
2. `v = SPECK64_128_decrypt(target, key)`。
3. 逆序撤销算术，所有加减均以 `& 0xffffffffffffffff` 保持 64 位回绕：

   ```python
   for op, c in reversed(ops):
       if op == "+": v = (v - c) & MASK64
       if op == "-": v = (v + c) & MASK64
       if op == "^": v = v ^ c
   ```

4. 按提取到的下标保存 `inputs[index] = v`，而不是按求解顺序追加。
5. 用 `v.to_bytes(8, "little")` 初始化 RC4，就地解密 `vmcode[next_level:]`；然后在新明文上解析下一层。

例如随后可恢复 `inputs[1] = 0xbf11b34d0ce941cc`，再由它解开下一组代码；总 WP 还展示了 `inputs[20] = 0xef320f9e6ae31520`。按相同方法完成 50 层后，以十进制逐行提交完整输入数组，VM 会进入末尾的输出指令并打印：

```text
RCTF{VM_ALU_SMC_RC4_SPECK!_593eb6079d2da6c187ed462b033fee34}
```

## 方法总结

本题需要同时识别三类结构：自定义 VM、由位运算展开的 ALU 宏，以及 RC4 驱动的分层 SMC。最有效的抽象单位不是单条 VM 指令，而是“装载输入—算术宏链—SPECK 比较—RC4 解下一层”这一固定模板。求解时必须保留三个细节：SPECK-64/128 为 27 轮、所有算术在 `2^64` 上回绕、RC4 密钥是输入整数的 8 字节小端表示；任一端序或位宽处理错误都会使下一层无法正常反汇编。
