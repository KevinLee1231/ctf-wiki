## 题目简述

附件是一个 64 位 Windows 校验程序。程序要求输入长度为 42、前缀为 flag{、末尾为 }，随后用相邻字符的乘加结果和前一项异或形成校验数组。反编译结果中的循环被编译器展开，但实际关系仍是一条可逆的递推链。

## 解题过程

先从主函数恢复约束。设花括号中的 36 个字符及右花括号依次为 $u_0,u_1,\ldots,u_{36}$，其中 $u_{36}=125$。程序使用的初始常量为 $a_i$，最终比较数组为 $e_i$，则校验关系为：

$$
e_0=u_0(u_1+30)-a_0
$$

$$
e_i=e_{i-1}\oplus\left(u_i(u_{i+1}+30)-a_i\right),\quad 1\le i\le35
$$

因为结尾的右花括号已知，可以从 $i=35$ 向前逐字节反推：

$$
u_i=\frac{(e_i\oplus e_{i-1})+a_i}{u_{i+1}+30}
$$

最后再由第一项求出 $u_0$。解题脚本如下：

~~~python
arr = [
    182, 265, 544, 82, 268, 608, 447, 293, 93, 234, 146, 40,
    126, 607, 181, 412, 369, 737, 466, 261, 74, 480, 468, 218,
    197, 86, 73, 556, 316, 112, 334, 449, 637, 363, 655, 97,
]
enc = [
    3885, 8151, 1285, 7055, 3323, 6023, 6806, 6048, 387, 8091,
    4191, 2603, 5938, 6969, 5322, 3246, 4979, 7841, 2575, 5420,
    3746, 16164, 9225, 10559, 14648, 10402, 14473, 13951, 11595,
    3371, 6015, 9974, 5721, 14001, 14462, 9425,
]

uuid = [0] * 36 + [ord("}")]
for i in range(35, 0, -1):
    numerator = (enc[i] ^ enc[i - 1]) + arr[i]
    denominator = uuid[i + 1] + 30
    assert numerator % denominator == 0
    uuid[i] = numerator // denominator

uuid[0] = (enc[0] + arr[0]) // (uuid[1] + 30)
flag = "flag{" + "".join(map(chr, uuid))
print(flag)
~~~

输出为：

~~~text
flag{159d1b01-b34e-44b9-b6fc-28438d6ede90}
~~~

## 方法总结

- 核心方法：把展开后的校验逻辑还原为相邻字符递推式，再利用已知右花括号从后向前求解。
- 识别特征：输入具有固定前后缀，当前校验项依赖相邻字符和上一校验项，且所有常量均可从二进制中提取。
- 注意事项：反编译器展示的循环展开不代表算法复杂；计算时应使用整数除法并检查整除性，以便及时发现常量或下标抄错。
