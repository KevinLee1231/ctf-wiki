# UMDCTF 2025 - top-ten-craziest-x86-instructions

## 题目简述

程序要求输入长度为 72、以 `UMDCTF{` 开头并以 `}\0` 结尾，随后用多组少见的
x86 SIMD/BMI 指令校验中间 64 字节。主要指令包括 `PCMPESTR*`、`PCLMULQDQ`、
`PEXT`、`ADDSUBPS`、`DPPS`、`PMADDWD`、`MPSADBW`、`PHMINPOSUW` 和
`RDSEED`。

正确做法不是爆破 64 个可打印字符，而是按四个 16 字节块分别翻译指令语义，再使用
GF(2) 多项式分解、浮点方程和 SMT 约束求解。

## 解题过程

### 1. 第一块：逆转 carry-less multiplication

`PCLMULQDQ` 把两个 64 位值视为 GF(2) 多项式并做无进位乘法。第一块的高、低
64 位相乘后必须等于给定的 128 位常量：

```c
res = _mm_clmulepi64_si128(block0, block0, 0x10);
```

把目标的每一位解释成 GF(2) 多项式系数，分解该多项式，再把不可约因子分配到两个
次数小于 64 的因子中。只保留能解码成 `!` 到 `~` 的组合。乘法本身无法区分两个
因子的次序，额外的：

```c
pext(low64, high64) == 562026407
```

负责选出唯一排列，恢复第 1 个 16 字节块。

### 2. 第二块：解四个 `float`

把第二块解释成小端的 `float f0,f1,f2,f3`。经过 `PEXT` 重排后，
`ADDSUBPS` 给出：

```text
r0 = f0 - f2
r1 = f1 + f0
r2 = f2 - f1
r3 = 2*f3
```

`DPPS` 又给出 `dp = 2*f1*f3`。于是可直接反解：

```python
f3 = r3 / 2
f1 = dp / (2 * f3)
f2 = r2 + f1
f0 = r1 - f1
```

按 IEEE-754 单精度重新打包，得到第二个 16 字节块。

### 3. 第三块：把 SIMD 校验翻译为 Z3 方程

`PMADDWD` 对前 32 字节与后 32 字节的对应 16 位字做相邻乘积求和。其低 128 位
恰好连接已恢复的第一块与待求第三块：

```text
out[i] = known[2i] * x[2i] + known[2i+1] * x[2i+1]
```

目标数组的低 8 个半字组成 4 个 32 位结果，因此这里得到 4 条乘积和方程。
`MPSADBW` 又对第三块的滑动 4 字节窗口与两个常量窗口：

```text
ef be ad de
de c0 37 13
```

计算绝对差之和。为 16 个未知可打印字节建立 32 位 BitVec，把 4 条乘积和方程与
16 条 SAD 方程一起交给 Z3，模型唯一，即得到第三块。

### 4. 第四块：再次分解并用最小元素定位

最后一块再次经过 `PCLMULQDQ`，所以沿用第一块的 GF(2) 因式分解。额外条件来自
`PHMINPOSUW`：把该块看成 8 个无符号 16 位数时，最小值必须是 `13156`，下标必须
为 5。这个条件筛掉其余因子排列；恢复后还可用 `PMADDWD` 结果的高 128 位验证
第二块与第四块之间的 4 条乘积和关系。

`RDSEED` 的两处循环只是等到随机数低 8 位为 228、低 16 位等于上述最小值，然后
再检查同一常量；它会影响等待时间，却没有增加 flag 熵。四块拼接并补回格式后得到：

```text
UMDCTF{stop_st0p_STOP!_w3_n33d_t0_g0_B4ck...back_t0_R0cks_in_th3_d3sErt}
```

## 方法总结

本题考查的是把指令语义转成合适的数学模型。无进位乘法应转到 GF(2) 多项式分解，
浮点向量操作应列方程并按单精度舍入复原，乘积和与 SAD 适合交给 SMT，最小值指令
则作为候选过滤条件。遇到冷门 intrinsic 时，先确认操作数宽度、lane 顺序、立即数
选择和有无符号语义，再决定求逆工具；把所有指令一股脑符号执行反而更难。
