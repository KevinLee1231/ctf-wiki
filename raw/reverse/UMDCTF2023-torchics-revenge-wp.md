# Torchic's Revenge

## 题目简述

题目提供一个体积很大的 Rust 可执行文件，运行后只打印无关文本。二进制静态链接了 `tch`（PyTorch 的 Rust 绑定），并嵌入一组序列化模型参数和一个 420 维输入；真正输出需要从编译产物中还原网络结构、权重与输入，再离线执行推理。

## 解题过程

反编译主函数可识别 `serde_pickle`、`tch::nn::linear`、激活函数和
`Module::forward`。线性层参数中的维度常量还原出网络：

```python
model = nn.Sequential(
    nn.Linear(420, 6969),
    nn.ReLU(),
    nn.Linear(6969, 3000),
    nn.Sigmoid(),
    nn.Linear(3000, 1337),
    nn.ReLU(),
    nn.Linear(1337, 8008),
    nn.ReLU(),
    nn.Linear(8008, 38),
    nn.ReLU(),
)
```

用 `binwalk` 或节区/魔数搜索可以定位嵌入的 pickle 数据。未知 pickle 可能在反序列化时执行代码，应先在隔离环境中检查并提取原始对象，而不是直接在宿主 Python 中加载。

序列化列表共有十项，偶数项是权重，奇数项是偏置。将字节数除以
`float32` 的 4 字节后，尺寸分别匹配：

```text
6969*420, 6969
3000*6969, 3000
1337*3000, 1337
8008*1337, 8008
38*8008, 38
```

按上述形状重建张量并依次赋给每层的 `weight`、`bias`。再从二进制中提取传给
`Tensor::of_slice` 的 420 个浮点数作为输入，执行前向传播。最后一层有 38 个输出，分别接近目标字符的 ASCII 值；四舍五入后连接为：

```text
UMDCTF{i_love_revving_torch_binaries!}
```

仓库未保留赛事发布的大型二进制和嵌入模型，仅保留完整官方还原说明与训练源码；上述结构、张量尺寸和解码方式均可由这些材料交叉验证。

## 方法总结

- 核心技巧：从框架 API 调用和维度常量恢复模型拓扑，再用序列化对象大小验证权重与偏置顺序。
- 巨大的二进制尺寸与 `serde_pickle` 调用共同提示模型数据被直接嵌入。
- 输出维度等于 flag 长度，且输出接近可打印 ASCII，是将模型输出转字符的明确信号。
- 不应在非隔离环境直接加载未知 pickle；静态提取或受控容器更安全。
