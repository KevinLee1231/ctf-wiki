# week3Mirror

## 题目简述

程序带有 UPX 壳。脱壳后，第一阶段要求求出两个 64 位整数 `n1`、`n2`，使两组大整数关系同时成立；Z3 可直接得到唯一解。随后程序还有一层逐字节异或校验。

官方 PDF 只公开了第一阶段方程脚本，并把后续描述为“简单异或”，没有保留异或数组、最终 EXP 或 flag；仓库也只有 `Mirror.exe`。因此正文可以准确补全方程解，但不能从现有文字材料伪造最后一层常量。

## 解题过程

先按 UPX 特征脱壳，再优先查看汇编而不是被优化器扰乱的伪代码。第一阶段可整理为：

$$
1969444366^2+1820452491=310\times(310^2+n_1)+n_2
$$

$$
2963569549^2+1719772226=704\times(704^2+n_1)+n_2
$$

使用 Z3 求解，并分别约束两个变量：

```python
from z3 import Ints, Solver, sat

n1, n2 = Ints("n1 n2")
solver = Solver()

lower = -(1 << 64) + 1
upper = (1 << 64) - 1
solver.add(n1 >= lower, n1 <= upper)
solver.add(n2 >= lower, n2 <= upper)

solver.add(
    1969444366**2 + 1820452491
    == 310 * (310**2 + n1) + n2
)
solver.add(
    2963569549**2 + 1719772226
    == 704 * (704**2 + n1) + n2
)

assert solver.check() == sat
model = solver.model()
print(model[n1])
print(model[n2])
```

结果为：

```text
n1 = 12446785179114514
n2 = 20207707034304107
```

原 WP 的脚本重复约束了 `n1`，遗漏了 `n2` 的范围；本版已修正。将两个整数输入脱壳后的程序即可进入后续异或校验。要补完最终 flag，仍需从 `Mirror.exe` 的 `checkflag`/`encrypt` 路径中提取目标数组和异或顺序，再把这些挑战专属常量写入 WP。

## 方法总结

当前可验证的部分是“脱壳 → 从汇编整理两条方程 → 求出 `n1`、`n2`”。现存官方材料没有支持最终异或层的常量，故不应把通用 XOR 模板冒充完整解答；后续恢复时至少要补充比较数组、密钥/索引规则和最终输出三项证据。
