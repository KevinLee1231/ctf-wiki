# BabyASM

## 题目简述

题目直接给出一段 32 位 Intel 语法汇编。`data` 区有 44 个字节，`main` 中的两段循环原地修改这些数据，最后把首地址传给 `printf`。目标是准确还原循环边界和字节运算。

## 解题过程

第一段循环的条件是：

```asm
cmp DWORD PTR [esp+28], 21
jle L4
```

因此索引范围为 $0\le i\le21$，共 22 字节，每个字节加 28。写回目标是单字节，所以运算结果按模 $2^8$ 保留。

循环结束时计数器已经是 22，程序直接进入 `L5`。第二段循环执行：

```text
data[i] = data[i] XOR data[i - 22]
```

其末尾条件为 `i <= 42`，所以处理索引 22 至 42。最后一个字节 `data[43] = 125`，即字符 `}`，保持不变。

按汇编逐条翻译即可：

```python
data = [
    20, 92, 43, 69, 81, 73, 95, 23, 72, 22, 24,
    69, 25, 27, 22, 17, 23, 29, 24, 73, 17, 24,
    85, 27, 112, 76, 15, 92, 24, 1, 73, 84, 13, 81,
    12, 0, 84, 73, 82, 8, 82, 81, 76, 125,
]

for i in range(22):
    data[i] = (data[i] + 28) & 0xff

for i in range(22, 43):
    data[i] ^= data[i - 22]

print(bytes(data).decode())
```

输出为：

```text
0xGame{3d24a572-394e-4ec7-b9c2-f9097fda1f4a}
```

也可以把文本保存为 `.s` 后用支持 32 位目标的 GCC 编译运行，但直接翻译汇编更容易看清循环范围，也不依赖本机是否安装 32 位链接库。

## 方法总结

这类短汇编题的重点是数据宽度和跳转边界：`jle` 使上下界都包含在循环中，`mov BYTE PTR` 又意味着加法只保留低 8 位。无需依赖反编译器，只要记录计数器、源索引和目标索引，就能稳定还原算法。
