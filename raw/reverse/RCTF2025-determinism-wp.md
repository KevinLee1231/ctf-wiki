# Determinism

## 题目简述

程序接收一个只含十进制数字的 `path.txt`，随后从标准输入读取 17 字节候选 flag。它把数万段加密的微型机器码函数动态解密、映射为可执行内存并依次调用：一类 `Code` 节点累加权重并约束 flag，另一类 `Edge` 节点根据路径字符选择两个后继之一。整体可还原为一棵稀疏二叉树；需要先找出满足“节点和 + 路径哈希”的唯一记录区间，再收集该区间上的字符约束交给 Z3。

总 PDF 没有收录本题；以下内容依据仓库中的单题 WP、生成源码和求解脚本补全。

## 解题过程

### 明确输入与动态代码机制

程序的调用方式为：

```text
./Determinism path.txt
```

路径文件会忽略换行，最多读取 1023 个字符，且每个字符必须是 `0` 至 `9`。随后程序要求候选 flag 长度恰好为 17，并限制为可打印字节。

主循环维护一个 `Controller`：

```c
state; check; sum; last_block_id; step_index;
path_buf[256]; path_buf_len;
```

每轮先从 `announce[last_block_id-1]` 取出 Code 密文，以 TEA-ECB 解密；再从 `g_edges[last_block_id-1]` 取出 Edge 密文，以 AES-128-ECB 解密。两类明文都会通过 `mmap`/`mprotect` 放进可执行页并作为函数调用。16 字节密钥在程序中逐字节异或 `0xff` 存放，恢复后分别是：

```text
Code/TEA key = Jump_h1gh_2_sky!
Edge/AES key = L1nk_2_th3_IN3T!
```

仓库单题 WP 对应的样本包含 58264 个 Code 表项和 26057 个 Edge 表项。用 IDAPython 遍历指针表、按各自长度解密，再反汇编明文函数即可离线处理，不需要让主程序执行数万次动态映射。

### 抽象成带权二叉树

Code 函数的模板负责三件事：在记录状态下把一个有符号权重加到 `controller.sum`；把该权重追加进 `path_buf`；按节点内嵌的表达式检查若干 flag 字节。Edge 函数则内嵌两个后继节点编号，并使用路径字符的低两位：

- bit 0 为 1：从未记录状态进入记录状态，或保持记录状态；
- bit 0 从 1 变为 0：把状态切为完成；
- bit 1：选择两个后继中的一条。

ASCII `'0'`、`'1'`、`'2'`、`'3'` 的低两位正好分别是 `00`、`01`、`10`、`11`，因此这四个字符同时编码“是否记录”和“走左/右分支”。

从每个 Code 明文提取权重，从每个 Edge 明文提取两个后继编号，便可构建稀疏二叉树。先 DFS 枚举所有向下的连续区间，筛选节点权重和等于全局目标的候选；再从根节点 DFS 到候选区间起点，为前缀选择 bit0=0 的路径字符。目标区间有 16 个节点，其权重依次为：

```text
5783, -7171, -1733, -3415, 7703, -8204, 2875, 7495,
7641, -183, -7455, 3473, 2466, -17, 2740, -5885
```

程序还对记录下来的权重数组执行 64 位 FNV-1a，并要求结果为 `6924299950838996780`，可用来排除同和路径。最终路径文件内容为：

```text
000011113333131333332
```

其中前四个 `0` 只负责从根走到记录区间起点，中间 16 个 bit0=1 的字符记录上述节点，末尾 `2` 令状态结束。

### 收集字符约束

沿该路径反汇编 16 个 Code 函数，再加上主函数中的最后检查，可得到完整的 17 字节约束。令 `f(i)` 表示第 `i` 个 8 位字符，`low8(x)` 表示结果低 8 位；双字符散列为：

```text
k = (a << 8) | b
k = (k + 0x9e3779b9) mod 2^32
k = (k ^ (k >> 16)) * 0x85ebca6b mod 2^32
k = (k ^ (k >> 13)) * 0xc2b2ae35 mod 2^32
hash2(a,b) = low16(k ^ (k >> 16))
```

转成 Z3 后加入可打印范围及以下等式：

```text
hash2(f0,  f2)  = 50141
hash2(f0,  f13) = 58957
hash2(f11, f2)  = 55732
hash2(f10, f13) = 5086
hash2(f6,  f15) = 34907

low8((f0 << 1) + (f4 >> 1) ^ f14)       = 208
low8(f14 ^ f2)                            = 84
low8(f0 * (f7 + 3) ^ f16 * 2)            = 145
low8((f1 ^ f2) + f16 * 2)                = 13
low8((f11 * 3 ^ f1 + f16) + 17)          = 241
low8(f10 * (f15 + 3) ^ f11 * 2)          = 100
low8(f8 ^ f1)                             = 3
low8(f4 * (f7 + 3) ^ f16 * 2)            = 37
low8((f4 ^ f10) + f5 * 2)                = 247
low8((f9 * 3 ^ f6 + f2) + 17)            = 113
low8(f3 * (f8 + 3) ^ f5 * 2)             = 180
low8((f12 << 1) + (f9 >> 1) ^ f6)        = 217
low8(f0 ^ f7)                             = 115
low8((f2 * 3 ^ f7 + f10) + 17)           = 72
```

这些表达式必须保持 8/16/32 位的原始截断语义；尤其右移应使用逻辑右移，乘加自然在固定位宽回绕。Z3 给出的唯一可打印解为：

```text
Ct1l_fL0w_th3_eNd
```

按题目说明补上外层格式，提交：

```text
flag{Ct1l_fL0w_th3_eNd}
```

## 方法总结

本题表面是海量动态解密代码，真正结构却很规整：Code 是“带权节点 + 字符约束”，Edge 是“状态位 + 二叉后继”。先离线解密并抽象图结构，再用两次 DFS 确定唯一记录区间，最后只把选中路径上的约束交给 Z3，能避免对数万函数逐个求解。实现时最容易出错的是把路径数字当数值而忘记程序传入的是 ASCII 字节，以及在 Z3 中丢失原始位宽、逻辑右移和低位截断。
