# week2re1

## 题目简述

附件给出一段汇编和静态字节数组。核心函数从倒数第二个字节开始向前遍历，用后一位已经恢复的字符和当前位置下标还原当前字符。

## 解题过程

汇编先扫描到字符串结尾，然后把循环下标设为 length - 2。循环体可化简为：

~~~c
void decode(unsigned char *buf)
{
    int length = strlen((char *)buf);
    for (int j = length - 2; j >= 0; --j) {
        buf[j] = buf[j] - buf[j + 1] + j;
    }
}
~~~

必须从后向前执行，因为计算第 $j$ 位时使用的 buf[j + 1] 已经是明文；数组末尾的右花括号作为递推初值保持不变。提取字节数组后直接模拟：

~~~python
enc = [
    210, 204, 198, 223, 171, 104, 149, 192, 148, 103, 103, 94,
    82, 86, 140, 136, 136, 129, 79, 84, 83, 80, 72, 119, 121,
    79, 87, 75, 70, 126, 121, 75, 73, 115, 122, 78, 77, 71,
    65, 67, 140, 125,
]

for j in range(len(enc) - 2, -1, -1):
    enc[j] = (enc[j] - enc[j + 1] + j) & 0xff

print(bytes(enc).decode())
~~~

输出为：

~~~text
flag{49be7981-6d3e-4341-a089-5f190d898437}
~~~

原 PDF 恢复出的 C 伪代码曾把起始下标写成 length - 1，但汇编中的 sub eax, 2 和其 Python 脚本都表明正确值是 length - 2；否则会先破坏末尾的右花括号。

## 方法总结

- 核心方法：把汇编循环还原成尾到头的差分递推，并用已知末字符作为初值。
- 识别特征：当前元素会减去后一元素，循环方向从字符串尾部向前，静态数组末尾已经是可见字符。
- 注意事项：循环边界比算术本身更关键；应让伪代码、汇编跳转和验证脚本三者互相校验，并按无符号字节处理溢出。
