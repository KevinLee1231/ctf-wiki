# EasyXor

## 题目简述

附件没有扩展名。文件识别结果表明它是 Debian 环境下由 GCC 编译的 64 位 ELF。程序内置一组密文字节和循环密钥 `raputa0xGame2025`，逐字节执行“异或后再叠加索引”的变换。

## 解题过程

未知文件应先根据魔数和头部确定格式，再选择分析工具。DIE 的识别结果可直接转写为：

```text
格式：ELF64
系统：Debian Linux（AMD64，64 位，DYN）
编译器：GCC 12.2.0
库：glibc 2.34
```

在 IDA 中跟踪校验循环可得加密关系：先将明文字节与循环密钥字节异或，再加上当前位置索引 $i$。因此逆变换为：

$$
p_i=(c_i-i)\oplus k_{i\bmod |k|}
$$

```python
enc = [
    0x42,0x1A,0x39,0x17,0x1D,0x9,0x51,0x55,0x2C,0x5F,0x63,0xC,0xD,0x16,0x62,0x27,
    0x55,0x64,0x55,0x26,0x6D,0x6A,0x18,0x34,0x88,0x65,0x6E,0x1C,0x21,0x6E,0x3D,0x23,
    0x6A,0x25,0x6B,0x63,0x68,0x7E,0x77,0x75,0x9A,0x7D,0x39,0x43
]
key = 'raputa0xGame2025'
for i in range(len(enc)):
    print(chr((enc[i]-i)^ord(key[i % len(key)])),end='')
```

减去索引必须发生在异或之前；Python 中减法优先级高于按位异或，但显式写成 `(enc[i] - i) ^ ...` 更清楚。输出拼接后即为 flag。

## 方法总结

- 核心技巧：按加密操作的逆序恢复“索引加法 + 循环 XOR”。
- 识别信号：固定密文字节数组、短密钥循环使用、循环变量参与每字节运算。
- 复用要点：异或自逆不代表整条表达式可以原样再算一次；混合加减时必须按逆序撤销每一步，并确认整数宽度和溢出语义。
