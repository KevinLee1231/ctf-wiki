# the_akari

## 题目简述

题目给出一个压缩后仍很大的 Akari（Light Up）棋盘，解答格式为所有灯泡坐标组成的 JSON 数组。程序验证整张棋盘后，从 256 个固定位置读取灯泡状态，组成 AES-256 密钥并解密 `flag_enc.bin`。

棋盘解压后为 $10205\times18389$，不适合人工渲染，但规则本身可以直接编码为 SAT/SMT 约束。公开仓库没有附生成器或官方 solve，以下求解路线由验证器的全部判定条件直接归纳，AES 填充与 flag 前缀可用作最终模型校验。

## 解题过程

为每个空白格 $(r,c)$ 建立布尔变量 $x_{r,c}$，表示是否放灯。预先扫描每一行和每一列，为空白格编号其所在的最大水平、垂直可见段。

Akari 规则转为以下约束：

1. 每个水平段至多有一个灯；
2. 每个垂直段至多有一个灯；
3. 每个空白格所在的水平段或垂直段至少有一个灯，从而保证被照亮；
4. 对数字墙 `0` 到 `4`，上下左右相邻灯变量之和必须等于数字。

例如数字墙约束为：

$$
x_{r-1,c}+x_{r+1,c}+x_{r,c-1}+x_{r,c+1}=d.
$$

大量 `0` 会立即把相邻变量置假，再触发照明与编号墙约束继续传播。实现时应按由墙和可见段组成的连通分量增量求解，避免把数百万个彼此独立或已被单元传播确定的格子一次性交给求解器。将模型中为真的变量输出为 `[[r,c], ...]`，可通过原程序的完整 Akari 校验。

程序随后按以下位置读取密钥比特：

```python
bits = []
for i in range(128):
    bits.append(board[2][22 + i * 48] == "O")
for i in range(128):
    bits.append(board[23 + i * 40][1] == "O")
```

每 8 位在转整数前反转：

```python
key = bytes(
    int("".join(map(str, bits[i:i+8]))[::-1], 2)
    for i in range(0, 256, 8)
)
```

`flag_enc.bin` 的前 16 字节是 IV，余下部分是 AES-CBC 密文。用上述 32 字节密钥解密并移除 PKCS#7 填充，得到：

```text
UMDCTF{r3venge_of_the_pclmulqdq}
```

## 方法总结

超大棋盘的视觉规模是干扰，真正结构仍是局部布尔约束。用可见段表达“互不照射”和“必须被照亮”，再结合数字墙做传播与分块求解，可以避免逐格回溯。最后要严格复现固定坐标和每字节位序，否则即使棋盘解正确也无法得到 AES 密钥。
