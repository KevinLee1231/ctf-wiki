# 代码金丹

## 题目简述

这是基础静态分析题。程序把用户输入与二进制中硬编码的目标字符串直接比较，没有额外的编码、加密或运行时生成过程，因此只需定位 `main` 中的比较逻辑。

## 解题过程

用 IDA 打开程序并对 `main` 按 `F5` 反编译。沿输入函数的交叉引用向后查看，可以看到 `strcmp` 将用户输入直接与下面的常量比较：

```c
strcmp(input, "0xGame{620bbfcb-e56f-4e6d-8069-9587e066130a}");
```

因此提交该字符串即可。为了避免只依赖反编译器显示，还可以在字符串窗口中定位 `0xGame{`，查看其交叉引用并确认它确实位于成功分支的比较路径上。

## 方法总结

面对简单校验程序，先从导入函数、字符串和 `main` 的控制流入手。硬编码明文只有在它参与成功分支判断时才是有效答案；结合交叉引用核对数据用途，可以排除诱饵字符串。
