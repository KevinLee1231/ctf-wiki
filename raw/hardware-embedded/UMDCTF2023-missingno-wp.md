# MissingNo

## 题目简述

题目原本提供一份 `MissingNo.iq` 复数基带采样：正常广播被一段奇怪语音插入，提示
“Gee, queue arr ex?” 指向 Gqrx SDR。官方仓库没有保留 IQ 文件，只留下外部附件入口、采样率和极简官方解法。因此以下内容是可由开发者记录确认的预期流程；中心频率、采样格式和瀑布图位置无法从当前源码树复测，不作臆测。

## 解题过程

在 Gqrx 中选择 IQ 文件作为输入源，将输入采样率设为：

```text
1.8e6 samples/s
```

启动接收后观察瀑布图，定位与正常节目不同的窄带语音信号。调谐到该信号并选择能清晰还原语音的解调模式；官方记录只说明“listen to the hex”，所以应以实际可懂度判断 AM/FM 模式，而不能在缺失样本时杜撰具体参数。

插入语音逐字读出一串十六进制。完整抄录后去掉空格和停顿，再按字节转成 ASCII：

```python
spoken_hex = "..."  # 从语音中逐字抄录
spoken_hex = "".join(spoken_hex.split())
print(bytes.fromhex(spoken_hex).decode())
```

官方记录确认解码结果为：

```text
UMDCTF{s0m3t1ng_l135_b3l0w}
```

## 方法总结

- 核心技巧：以 1.8 MS/s 正确载入 IQ 采样，在瀑布图中定位插入语音，解调后把口述十六进制还原为 ASCII。
- 识别信号：附件扩展名为 `.iq`、题面直接影射 Gqrx，并描述“广播被声音打断”时，应走 SDR 解调而非把文件当普通 WAV。
- 复用要点：采样率、复数格式、端序和解调模式必须与样本一致；本仓库缺失原始 IQ，不能把未验证的中心频率或调制方式写成事实。
