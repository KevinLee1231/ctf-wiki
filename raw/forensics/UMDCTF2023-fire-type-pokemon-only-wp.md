# Fire Type Pokemon Only

## 题目简述

附件是一份网络抓包。题目模拟一台暴露到互联网的主机被通过 FTP 访问：FTP 控制通道以明文传输用户名、密码和命令，数据通道中还传输了一个弱口令保护的 ZIP。目标是从既有流量中恢复登录凭据和传输文件。

## 解题过程

先在 Wireshark 中使用过滤器：

```text
ftp
```

或只查看请求命令：

```text
ftp.request.command
```

在控制连接中可以直接看到 `USER` 和 `PASS`。FTP 本身不加密这些字段，因此无需破解认证哈希。

随后选择 Wireshark 的“文件 → 导出对象 → FTP-DATA”，导出数据连接上传或下载的 ZIP。也可以按
`ftp-data` 过滤器定位对应 TCP 流并重组载荷。

ZIP 使用的弱口令与抓包中 FTP 登录口令相同。将从 `PASS` 命令恢复的字符串用于解压，即可读取其中的 flag：

```text
UMDCTF{its_n0t_p1kachu!!}
```

本题的两步证据必须关联起来：控制通道给出口令，数据通道给出受保护对象。只导出文件而忽略登录会话，会把一个本可直接恢复的口令误当成离线爆破题。

## 方法总结

- 核心技巧：分别重组 FTP 控制流和 FTP-DATA 对象，再复用控制通道明文口令解开传输档案。
- 识别信号：PCAP 中出现 FTP、`USER`、`PASS` 和独立数据连接时，优先进行协议对象导出，不要从原始包字节手工拼接。
- 复用要点：FTP 登录口令、ZIP 口令和文件内容属于不同证据层；保留会话端点和时间顺序，才能可靠地把凭据关联到正确对象。
