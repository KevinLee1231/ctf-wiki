# Vigenère XOR

## 题目简述

题目把一段较长的英文逐字节与循环密钥异或，本质上是 byte alphabet 上的 Vigenère/repeating-key XOR：

$$
C_i=P_i\oplus K_{i\bmod L}.
$$

密钥长度 $L$ 被隐藏，但长英文提供了足够的周期统计量，可先用重合指数确定周期，再逐列恢复密钥字节。

## 解题过程

对每个候选长度 $L$，把密文按 $L$ 分组，并将相同下标的字节组成一列。正确 $L$ 下，每列都是同一个单字节 XOR 加密的英文字符分布，重合指数会接近英文的约 $0.0686$；错误长度更接近均匀随机分布。

扫描 1 到 254，并避免把真周期的整数倍误当新候选，得到：

```text
key length = 28
```

然后对第 $j$ 列枚举 $k_j\in[0,255]$。保留解密后全部为可打印 ASCII、空格数量合理且小写字母占比高的候选。每列都得到唯一字节，拼出：

```text
key (hex) = 7a388220923334962181fb6d576e4acd3d608e3c722427859c63fa2
```

循环使用该密钥解密：

```python
plaintext = bytes(
    c ^ key[i % len(key)]
    for i, c in enumerate(ciphertext)
)
```

长英文正文中直接出现：

```text
UMDCTF{d1d_y0u_use_k4s!sk1_0r_IoC???}
```

## 方法总结

- repeating-key XOR 可按密钥位置转置为若干独立的单字节 XOR 问题。
- Kasiski 检验适合寻找重复片段间距，重合指数适合对候选周期做统计排序；长文本会显著提高可靠性。
- 真密钥长度的倍数也可能表现出高重合指数，应优先选择能解释数据的最小基本周期。
- 逐列评分不应只检查可打印字符，还应加入空格、英文频率或语言模型分数，以降低伪候选。
