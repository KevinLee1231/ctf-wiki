# 打好基础

## 题目简述

题面只有 `ENJOY HGAME！`，附件是一段 emoji 文本。文本外观是高密度 emoji 序列，符合 base100 一类编码的输入形态；题目没有其他服务或二进制附件。长期 WP 中只需要保留数据形态和开头样例，不必全文保存整段 emoji。

## 解题过程

原始数据是一长串 emoji，外观符合 base100 编码，开头形如：

```text
🐲👝🐱👁👠👮👌👞👂🐬🐽🐡👎👮🐪🐿...
```

解码顺序如下，每一层输出仍是下一层可识别的编码文本：

```text
base100 -> base92 -> base91 -> base85 -> base64 -> base62 -> base58 -> base45 -> base32
```

最终结果：

```text
hgame{L4y_a_sO11d_f0unDaTi0n}
```

## 方法总结

杂项编码题先识别最外层数据形态，再按输出是否仍像编码文本继续剥离。遇到 emoji 大段文本优先考虑 base100；每层解码后都检查字符集和 padding，而不是盲目套所有编码。
