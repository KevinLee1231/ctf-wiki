# DFA Speck 2

## 题目简述

第二问沿用第一问恢复的 `k32`，另给一组在倒数第二轮输入处注入的单比特故障密文。目标是继续恢复 `k31`、逆转 Speck 128/128 密钥扩展，并解密原始 128 位明文。

题目特别规定明文由两个 64 位字按小端字节拼接，不能沿用故障密文的显示端序。

## 解题过程

第一问已经得到：

```text
k32 = 306e675f336e7566
```

先用 `k32` 对正确密文和第二组故障密文各逆一轮：

```python
def inverse_round(x, y, k):
    y = ror(y ^ x, 3)
    x = rol((x ^ k) - y, 8)
    return x & MASK64, y & MASK64
```

这样故障就重新落到“最后一轮输入前单比特翻转”的同一模型，可复用第一问的进位递推求出 `k31`。最高位仍有两个候选；正确分支为：

```text
k31 = 488bcae68d876541
```

Speck key schedule 中，下一轮的另一个状态字满足：

$$
j_{32}=\operatorname{ROL}_3(k_{31})\oplus k_{32}.
$$

由此得到：

```text
j32 = 7430306b5f555f6c
```

随后从 $(j_{32},k_{32})$ 逆执行轮密钥生成过程，恢复初始 128 位主密钥：

```text
07b3fb254b53de35 c07f5e080604d9f4
```

对原始密文解密，并按题目要求将两个 64 位明文字分别按小端转为字节后拼接：

```text
d0_yU_h8_us_y3t?
```

最终 flag：

```text
UMDCTF{d0_yU_h8_us_y3t?}
```

## 方法总结

- 多轮故障题通常是逐层剥离：先恢复末轮密钥，用它逆一轮，再把下一组样本化归同一个故障模型。
- `k31` 的最高位也有二义性，应把两个候选都带入 key schedule 和明文可读性验证。
- Speck 同时涉及“文件中两个字的顺序”“整数的显示端序”“明文字节端序”，每层都要单独确认。
- 明文可读不仅是提示，也是验证轮密钥候选和端序是否正确的强约束。
