# UMDCTF 2025 - brainrot13

## 题目简述

题目使用同一个 1024 位 RSA 模数和 $e=3$，分别加密原 flag 与整串文本经过 ROT13 后的结果。所谓“OAEP”并不是真正的 OAEP，而只是不断在消息末尾追加固定字节串 `OAEP`，直到长度达到 120 字节：

```python
def encrypt(b):
    while len(b) < 120:
        b += b"OAEP"
    return pow(bytes_to_long(b), 3, n)
```

flag 格式被限制为 `UMDCTF{[a-z]+}`，总长度固定为 28。两份明文因此存在可枚举的线性差值，可以使用 RSA 低指数相关消息攻击。

## 解题过程

设追加固定填充后的原消息为 $m$，ROT13 版本为 $m+\Delta$，则已知：

$$
c_1\equiv m^3\pmod n,\qquad
c_2\equiv (m+\Delta)^3\pmod n.
$$

ROT13 对每个英文字母只会让其字节值增加或减少 13。`UMDCTF{` 前缀的变化方向已知，20 个小写 flag 字符各有 $\pm13$ 两种可能，右花括号和固定后缀不变。因此只需枚举 $2^{20}$ 个差值，而不是枚举明文本身。

对每个候选 $\Delta$，在 $\mathbb Z_n[x]$ 中构造：

```python
f = x**3 - c1
g = (x + delta)**3 - c2
h = gcd(f, g)
```

正确差值下，两个多项式拥有公共根 $m$，其最大公因式会退化为一次多项式；错误差值通常得到 1。取该一次因子的根即可恢复完整填充消息，再截取 flag。

这就是 Franklin–Reiter 相关消息攻击在 $e=3$ 时的直接应用。固定、确定性的伪填充没有破坏两份明文之间的代数关系，反而让攻击者能够准确写出 $\Delta$。

最终恢复：

```text
UMDCTF{shouldverotatedtwice}
```

## 方法总结

- 核心技巧：枚举 ROT13 造成的逐字节 $\pm13$ 差值，再对两个 RSA 三次多项式求最大公因式。
- 识别信号：同模数、同低指数加密相关消息，且消息变换是可预测的线性字节差。
- 复用要点：固定字符串拼接不是安全填充；先把字符关系提升为大整数差值，再判断差值空间是否小到可以枚举。
