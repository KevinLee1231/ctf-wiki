# quant?

## 题目简述

服务端模拟 16 个输入量子比特和 1 个辅助量子比特，并隐藏一个 16 位目标串。提交内容是受限的 OpenQASM 风格程序，最多调用 oracle 250 次；服务端进行 512 次采样，只有目标态概率不低于 $0.55$ 才会给出 flag。

搜索空间大小为 $N=2^{16}$。经典枚举无法通过单次提交完成，而 Grover 搜索只需约 $\frac{\pi}{4}\sqrt N$ 次 oracle。

## 解题过程

先把辅助量子比特制备为 $|-\rangle$：

```qasm
x q[16];
h q[16];
```

再对 16 个输入量子比特施加 Hadamard 门，得到均匀叠加。oracle 以辅助比特为目标执行受控翻转；由于辅助比特处于 $|-\rangle$，相位回踢会把被标记输入态的振幅取反。

Grover 的最优迭代次数为

$$
r=\operatorname{round}\left(\frac{\pi}{4}\sqrt{2^{16}}\right)=201,
$$

没有超过 250 次限制。服务端已经提供 `diffuse` 指令，因此每轮只需：

```qasm
oracle q[0],q[1],q[2],q[3],q[4],q[5],q[6],q[7],q[8],q[9],q[10],q[11],q[12],q[13],q[14],q[15],q[16];
diffuse q[0],q[1],q[2],q[3],q[4],q[5],q[6],q[7],q[8],q[9],q[10],q[11],q[12],q[13],q[14],q[15];
```

重复 201 次后，将每个输入量子比特测量到同编号经典位：

```qasm
measure q[0] -> c[0];
// 依次测量到 q[15] -> c[15]
```

服务端采用小端量子位索引，但打印 bitstring 时会反转为题目中的自然顺序。正确程序会让隐藏态占据绝大部分概率，从而直接输出：

```text
UMDCTF{0110010101111010}
```

## 方法总结

本题是标准的黑盒唯一目标搜索。辅助比特的 $|-\rangle$ 状态把值 oracle 转为相位 oracle，随后交替执行 oracle 与关于均值的反射。关键参数是 201 次迭代：过少时放大不足，过多又会把振幅旋离目标态。
