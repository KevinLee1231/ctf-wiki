# EasyCurve

## 题目简述

本题曲线由 Pell 方程构造，题目只给出参数 `D`，没有给出 `u`。服务通过 OT 限制用户不能同时得到曲线上点的 `x` 和 `y`，预期是避免直接恢复 `u`；但曲线群与 `GF(p)` 之间存在映射，可把曲线 DLP 转成模 `p` 的 DLP。

题目源码和 exp 仓库中包含曲线参数、交互脚本和相关论文。题目机制是把 Pell 型曲线上的运算和 OT 结合：预期解通过构造 OT 查询获得足够关系，消去未知数后转成模 `p` 离散对数；因为 `p-1` 很光滑，DLP 可轻松求解。非预期是利用 `x/y` 过小，取足够大的 OT 参数同时恢复 `x` 和 `y`，直接得到 `u` 后在原曲线上求 DLP。

## 解题过程

题目源码、详细 exp 和参考文献见 [`shal10w/d3ctf2021_EasyCurve`](https://github.com/shal10w/d3ctf2021_EasyCurve)。该仓库中的关键思路已经转写为下文的 Pell 曲线群映射、OT 查询构造和 `GF(p)` DLP 化简。

题中曲线由 Pell 方程构造，但参数没给全，只给了 `D`，没给 `u`。同时 OT 也不允许用户同时拿到 `x` 和 `y` 来算出 `u`。研究曲线加法规则后，可以把原本用斜率描述的加法改写成坐标公式：

```text
A(x1, y1) + B(x2, y2) = C(x3, y3)
x3 = (x1*x2 + D*y1*y2) * inverse(u, p) mod p
y3 = (x1*y2 + x2*y1) * inverse(u, p) mod p
```

则两点相加后的坐标满足如下关系，这也是后续建立有限域映射的基础：

可以推出当D与u为二次剩余时,曲线上的点与GF(p) 有一个映射

```text
(x, y) -> a * (x - d*y), 其中 a^2 = u, d^2 = D
k*(x, y) -> (a * (x - d*y))^k
```

通过这个映射能够将曲线上的dlp问题转化为模p的dlp问题。
同时,题中实现的OT虽然不能同时得到x与y,但是可以通过构造

```text
v = (x0 + (-d)^e * x1) * inverse(1 + (-d)^e, n) mod n
```

来让 OT 返回的两个消息满足

```text
m0 - d*m1 = x - d*y
```

若 `A = eG`，则

```text
X_A - d*Y_A = a^(e-1) * (X_G - d*Y_G)^e
```

里面还有未知数 `a`，但题目给了三组数据，因此可以用两组数据相除来消去 `a`。最后题目转化为模 `p` 的 DLP；分解 `p-1` 可发现它很光滑，可以轻松计算出 `e`。

这题也存在非预期。由于 `x` 和 `y` 都过小，如果 OT 中选择的模数/参数大于实际坐标值，返回结果模回去后可以先得到 `x`，再由差值除出 `y`；得到完整点坐标后可恢复 `u`，随后直接在原曲线上计算 DLP。
题目源码与 exp 的关键依赖是：Pell 曲线群可映射到 `GF(p)` 中处理，OT 交互会泄露足够多的代数关系，配套论文用于说明该曲线群与有限域乘法群之间的转换方式。

## 方法总结

- 核心技巧：把特殊曲线群上的 DLP 通过代数映射降到 `GF(p)` 上，再利用 `p-1` 光滑求离散对数。
- 识别信号：Pell 方程构造曲线、加法规则可改写为点坐标关系、OT 试图隐藏同一点的两个坐标。
- 复用要点：自定义曲线题先研究群同构/映射，不要急着在原曲线上硬算 DLP；交互限制如果只靠取模或阈值，也要检查小坐标是否能被非预期完整恢复。
