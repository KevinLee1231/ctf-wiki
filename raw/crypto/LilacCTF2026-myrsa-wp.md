# myRSA

## 题目简述

题目给出一个三素数 RSA：`n = p * q * r`，`e = 65537`，并提供一个平方根 oracle。源码中可以看到 `p`、`q` 不是普通随机素数，而是由同一个参数 `pp` 生成：

```python
assert p == pp**2 + 3 * pp + 3 and q == pp**2 + 5 * pp + 7
```

oracle 对输入 `x` 分别在 `p`、`q`、`r` 下求平方根，再用 CRT 合并；但输入受限为 `80 < x < 100` 且不能是平方数。实际有效信息很少，核心突破点不是暴力利用 oracle，而是利用 `p` 的特殊代数结构，把它转化为一个 CM 椭圆曲线上的 ECM 型分解。

## 解题过程

### 关键观察

由 `p = pp^2 + 3pp + 3` 可得：

$$
4p = 4pp^2 + 12pp + 12 = (2pp + 3)^2 + 3
$$

这正好对应判别式 `D = 3` 的复乘椭圆曲线。对 CM 曲线，有 Frobenius trace 满足：

$$
4p = t^2 + D y^2
$$

本题中 `t = 2pp + 3`，`D = 3`，`y = 1`。`D = 3` 的 CM 曲线可写成：

$$
E: y^2 = x^3 + b
$$

由于只有 `oracle(96)` 能返回有效平方根，可以从 `96 = 4^3 + 32` 猜到可用曲线为：

$$
E: y^2 = x^3 + 32
$$

oracle 返回的是模 `n` 下点 `(4, y)` 的 `y` 坐标，因此可在 `Zmod(n)` 上构造点 `G`。

### 分解思路

在模 `p` 的曲线上，若 `E.order()` 是曲线阶，则：

$$
E.order() \cdot G = O \pmod p
$$

把同样的计算放到 `Zmod(n)` 中做，点乘结果的 `x` 坐标会带有 `p` 的因子，因此可以直接：

```python
E = EllipticCurve(Zmod(n), [0, 32])
G = E(4, oracle_96_y)
p = int(gcd((n * G)[0], n))
```

拿到 `p` 后，由一元二次方程恢复 `pp`：

```python
pp = var("pp")
solve(pp**2 + 3 * pp + 3 - p, pp)
```

再计算：

```python
q = pp**2 + 5 * pp + 7
r = n // (p * q)
d = inverse(65537, (p - 1) * (q - 1) * (r - 1))
m = pow(c, d, n)
```

完整求解流程就是：构造 `y^2 = x^3 + 32`，填入 `oracle(96)` 得到的点，求 `gcd((n * G)[0], n)` 分解 `n`，最后 RSA 解密。

### 验证

脚本恢复出 `p` 后，`p` 能满足 `p = pp^2 + 3pp + 3`，并且 `q = pp^2 + 5pp + 7` 可以整除 `n`。这同时验证了曲线分解结果和后续 RSA 私钥恢复。

## 方法总结

- 核心技巧：识别特殊素数结构，把 RSA 分解转化为 CM 椭圆曲线上的 ECM 型攻击。
- 识别信号：素数由二次多项式生成，且能整理成 `4p = t^2 + D y^2`，应联想到 CM 曲线和 Frobenius trace。
- 复用要点：oracle 本身可能只泄露一个点，但只要该点落在结构化曲线上，就可以通过曲线阶点乘让坐标携带素因子，再用 `gcd` 分解模数。
