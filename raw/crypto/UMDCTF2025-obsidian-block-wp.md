# UMDCTF 2025 - obsidian-block

## 题目简述

题目给出一个 256 位、24 轮的自定义密码以及一组已知明密文和 flag 密文。每轮只有两步：

```python
block = (block + key) % 2**256
block = rotr(block, round_constant[i])
```

同一个 256 位密钥在所有轮重复使用。循环旋转在模 $2^{256}-1$ 下等价于乘法，而模 $2^{256}$ 加法与它之间的差异只剩每轮一个进位位，因此可以枚举 24 个进位并解线性同余。

## 解题过程

令 $Q=2^{256}-1$。对 256 位整数，循环右移 $r$ 位满足：

$$
\operatorname{ROR}_r(x)\equiv 2^{256-r}x\pmod Q.
$$

一轮中的模 $2^{256}$ 加法可以写为：

$$
(x+k)\bmod 2^{256}=x+k-b_i2^{256},
$$

其中 $b_i\in\{0,1\}$ 是该轮是否溢出的进位。因为 $2^{256}\equiv1\pmod Q$，在模 $Q$ 下它只贡献一个 $-b_i$。

把 24 轮展开，记从第 $i$ 轮到末轮的旋转乘数积为 $R_i$，可得：

$$
c\equiv R_0p+\left(\sum_{i=0}^{23}R_i\right)k-\sum_{i=0}^{23}R_i b_i\pmod Q.
$$

已知 $p$ 与 $c$ 后，枚举全部 $2^{24}$ 个进位向量。对每个候选计算：

$$
d=c-R_0p+\sum R_i b_i,
\qquad
K=\sum R_i,
$$

并求解线性同余 $Kk\equiv d\pmod Q$。若 $g=\gcd(K,Q)$，只有 $g\mid d$ 时有解；先在模 $Q/g$ 下求逆，再枚举由公因数产生的 $g$ 个密钥同余类。

官方脚本会把每个候选密钥重新代回原始的模 $2^{256}$ 加密函数。只有真正匹配原已知密文的密钥才会保留，随后逆序执行左旋和减密钥：

```python
for i in range(23, -1, -1):
    block = rotl(block, round_constants[i])
    block = (block - key) % 2**256
```

解开第二个密文得到：

```text
UMDCTF{diamond_pickaxe_no_way!!}
```

## 方法总结

- 核心技巧：把循环旋转转成模 $2^w-1$ 的乘法，把模 $2^w$ 加法的非线性压缩成每轮一个可枚举进位位。
- 识别信号：重复轮密钥、仅由加法和固定旋转组成、轮数足以枚举进位状态但不足以提供安全性。
- 复用要点：在线性化后的模数中求出的只是候选同余类，必须回到原始位宽运算验证，避免忽略 $2^w$ 与 $2^w-1$ 表示差异。
