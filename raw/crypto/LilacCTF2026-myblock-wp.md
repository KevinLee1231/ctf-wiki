# myBlock

## 题目简述

题目实现了一个 8 轮 Feistel 结构，每轮子密钥为 16 bit，分组左右半块也都是 16 bit。轮函数工作在 `GF(2^16)` 上：

```python
POLY = 0x1002D
ROUNDS = 8

def enc_block(self, L, R):
    for k in self.subkeys:
        T = self.cube(L ^ k)
        new_R = R ^ T
        new_L = L
        L, R = new_R, new_L
    return L, R
```

服务允许选择明密文样本数量，返回随机明文及其密文，要求在 512 秒内恢复全部 8 个 16-bit 子密钥。关键点是轮函数 `F(x, k) = (x + k)^3` 可以写成多项式，因此明密文样本会给出关于子密钥的多元多项式约束。

## 解题过程

### 关键观察

因为域加法就是异或，乘法在 `GF(2^16)` 中由不可约多项式 `0x1002D` 约束，整个 Feistel 网络可以符号化为多项式映射。已知多组 `(P_L, P_R) -> (C_L, C_R)` 后，未知量只剩每轮 16-bit 子密钥。

原题的预期方向是插值攻击：先把若干轮正向展开、若干轮反向剥离，构造中间状态的多项式基，然后用样本线性化得到约束。官方 WP 中给出的求解器核心逻辑是：

1. 生成正向和反向可达的单项式指数集合。
2. 用明文样本建立正向矩阵，对矩阵做消元，得到左零空间约束。
3. 枚举当前待恢复的 16-bit 轮密钥，把密文反向剥一轮。
4. 检查反向矩阵秩是否异常下降，秩下降时说明枚举密钥正确。
5. 每恢复一轮就用该密钥更新密文状态，继续恢复下一轮。

### 求解步骤

正向/反向的基生成依赖 Feistel 轮函数的三次形式。若当前轮中出现 `(L + K)^3`，单项式次数会由原来的基三重组合得到：

```cpp
for (auto e1 : bases)
  for (auto e2 : bases)
    for (auto e3 : bases)
      T.insert({e1.first + e2.first, e1.second + e2.second});
```

实际利用时不用直接求完整 Groebner basis，而是用线性代数检测秩下降。对每个候选 `k`，先把密文反向剥一轮：

```cpp
L_in = R_out;
R_in = L_out ^ (R_out + k)^3;
```

然后把这些候选中间状态代入反向基矩阵。如果矩阵秩低于正常值，说明该 `k` 与正向约束兼容，即为当前轮密钥。重复 8 次后拼接所有 16-bit 子密钥，并与服务输出的 `sha3_256(key)` 校验。

### 验证

恢复出的 8 个子密钥按大端拼接为 16 字节，与服务打印的 `sha3_256(key)` 摘要一致后即可提交原始 key，服务会返回 flag。

## 方法总结

- 核心技巧：对低度 Feistel 轮函数做插值/线性化，通过中间状态约束和秩下降逐轮恢复密钥。
- 识别信号：轮函数是有限域上的低次多项式，服务给大量明密文样本，目标是恢复短子密钥。
- 复用要点：不一定要直接上 Groebner basis。遇到多轮低度结构时，可以分割轮数，用“正向空间约束 + 反向枚举检测”降低求解复杂度。
