# StegLab-PointAttack 1& 2

## 题目简述


原始方向为 `Steg`；归档时按主要障碍归入 `forensics`，因为解法主体是鲁棒图像隐写设计和攻击后恢复。

StegLab 是一个在线隐写算法评测平台。选手需要提交 `encrypt.py` 和 `decrypt.py`：前者把系统给出的短 key 写入若干张图片，后者在平台对图片进行攻击后再把 key 读回来。平台会先用 PSNR 判断嵌入后的图片是否和原图过于接近或破坏过大，随后运行攻击脚本并根据解密输出判定是否通过。

官方 writeup 参考：[StegLab 解答与心得记录以及问题反思](http://www.snowywar.top/?p=4211)。外链中的重要信息是：PointAttack 会在图像上随机绘制点和若干条直线，ToJPGAttack 会把图片转成 JPEG 再伪装回 PNG。因此解法不能只依赖单个像素的精确 LSB，而要考虑扰动、压缩和冗余恢复。

## 解题过程

### PointAttack

PointAttack 的攻击脚本会在原图上随机画线并散布大量随机点。原始 LSB 只要被这些点覆盖就可能损坏，所以可行思路是增加冗余：把同一 bit 或同一字符写入多个位置，解密时用多数投票、通道统计或固定随机种子定位来恢复。

一种基础做法是把 key 转成二进制，按固定顺序或固定 PRNG 种子写入像素最低位；解密时读取相同位置，并按 8 bit 一组转回字符。为了防止 PointAttack 污染局部像素，需要给每个 bit 分配多处写入点，或者在 RGB 三通道中重复嵌入后投票。

### ToJPGAttack

ToJPGAttack 的关键是 JPEG 有损压缩会破坏低位信息，直接 LSB 几乎不可用。外链中给出的有效方向是把信息写成更强的空间域特征，例如在固定随机位置上把像素改成明显的黑/白或亮/暗状态，并在解密时比较该点和周围邻域的亮度差异。这样即使经历 JPEG 压缩，强对比特征仍然有机会保留下来。

处理流程可以概括为：

1. 将 key 映射为有限字符集编码，例如每个字符 6 bit。
2. 使用固定种子选择嵌入坐标，保证加密和解密访问同一批点。
3. 对每个 bit 写入强特征，`1` 写成高亮点，`0` 写成暗点，或者在局部邻域中制造可统计的差异。
4. 解密时读取相同坐标，结合周围像素均值判断该位置代表 `0` 还是 `1`，再还原字符。

这种做法比普通 LSB 更适合 JPEG 攻击，因为判定依据是局部亮度/颜色差异，而不是单个像素最低位。

## 方法总结

- 核心技巧：把隐写从“脆弱的单点 LSB”改成“带冗余或强特征的鲁棒嵌入”。
- 识别信号：题目存在 `attack.py`、PSNR 阈值、平台反复攻击图片时，应先分析攻击脚本破坏的是点、线、压缩还是格式转换。
- 复用要点：PointAttack 适合冗余和多数投票；JPEG 类攻击要避免直接依赖最低位，改用固定坐标、邻域统计、亮暗特征或频域特征。
