# week1垃圾邮件

## 题目简述

附件包含 `key.txt` 和 `flag.txt`。前者通过多层编码隐藏口令，后者是需要该口令才能提取载荷的文本隐写文件；两条线索必须串联使用。

## 解题过程

先处理 `key.txt`。每一层都可由字符集识别下一步格式，完整顺序为：

```text
AAencode → Brainfuck → Base64 → Base64 → Base58 → Base32
```

逐层解码后得到口令：

```text
P@33w0rD
```

再检查 `flag.txt` 的原始字节、空白字符和文件格式，选择与载体匹配的文本隐写解码器，并启用“带密码/加密载荷”模式，输入上述口令提取 flag。这里口令不是压缩包密码，而是隐写算法用于解密隐藏数据的参数。

原仓库没有保留 `key.txt`、`flag.txt` 或具体工具截图，因此不能仅凭“文本隐写”四个字确定唯一算法。原题解中“上网找网站”的外部依赖不应保留；可复现时应先依据文件字节特征确认载体格式，再选择实现该格式的本地工具或解码器。

## 方法总结

- 核心方法：先沿编码链恢复口令，再用口令解开文本隐写载荷。
- 识别特征：AAencode 具有固定 JavaScript 外观，Brainfuck 主要由八种符号组成，Base64/Base58/Base32 则可按各自字符集逐层判断。
- 注意事项：不要跳过中间结果验证；当原始载体缺失时，不能把某个在线网站名称当成隐写算法本身，更不能臆造最终 flag。
