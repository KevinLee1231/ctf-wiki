# SUCTF2026-Artifact_Online

## 题目简述

题目提供一份符文长文本和一个在线“artifact”。连接服务后先完成 SHA-256 PoW，随后会看到一个由符文组成的 5×5 方块正面与右面，并可选择 `Twist` 或 `Activate`：前者旋转六面方块，后者在正面选取字符并把结果作为 Linux shell 命令执行。

题目提示为：

```text
Each rune represents a character.
Try to craft some commands to find the secret outside the current directory.
The flag is not located in the current directory.
```

服务进入主界面后只有 40 秒，且每次连接都会重新打乱状态，所以不能靠人工逐格尝试。核心任务是先恢复符文映射，再把所有旋转建模为本地置换，搜索出能在激活面组成目标命令的操作序列。

## 解题过程

### 1. 恢复符文替换表

附件 `something mysterious.txt` 并非随机字符。统计后可见只有约 31 种高频符号，并保留了空格、标点和分行结构；出现最多的 `ᛨ` 对应空格，因此它是一个单表替换密码。

用词频、词形和引号位置做初步替换后，可识别出明文来自 Robert A. Heinlein 的短篇小说 *All You Zombies*，对应片段以 “The Unmarried Mother ...” 开头。将找到的原文与符文逐字符对齐，即可恢复完整映射：

```text
a ᚠ  b ᚢ  c ᚦ  d ᚨ  e ᚱ  f ᚲ  g ᚷ  h ᚹ  i ᚺ  j ᚾ  k ᛁ  l ᛃ  m ᛇ
n ᛈ  o ᛉ  p ᛋ  q ᛏ  r ᛒ  s ᛖ  t ᛗ  u ᛚ  v ᛜ  w ᛟ  x ᛞ  y ᛣ  z ᛤ
, ᛥ  ; ᛦ  . ᛧ  空格 ᛨ  ? ᛩ  { ᚪ  } ᚫ  - ᚬ  " ᚭ  ! ᚮ  ' ᚯ
```

这张表既用于把目标命令编码为符文，也用于解码命令执行结果。附件的作用是提供足够长的已知明文样本，不需要把整篇小说或整段密文复制进求解器。

### 2. 明确交互规则

PoW 要求寻找字符串 `S`，使：

```text
sha256(prefix + S).hexdigest()[:6] == "000000"
```

通过后进入两种模式：

- `Twist`：可执行 `R1`–`R5`、`C1`–`C5`、`F1`–`F5`，后缀 `'` 表示逆旋转。`R`、`C`、`F` 分别绕三条轴旋转指定层，每个操作执行四次会回到原状态。
- `Activate`：锁定当前正面，光标从左上角开始，第一次只能在第 0 行横向移动；按 Enter 取字符后，允许移动的方向在纵向和横向之间交替。按 `x` 后，已选字符会被拼成命令并执行。

源码进一步说明，命令通过 `subprocess.run(..., shell=True)` 以低权限用户运行，工作目录固定为 `/home/ctf`，输出再用同一符文表编码。因此这里不是猜“魔法单词”，而是在受空间路径约束的字符板上构造 shell 命令。

### 3. 采集六个面并建立本地模型

远端只同时显示正面和右面。进入 `Twist` 后，对各层执行已知旋转并记录每次的两张可见面，即可把后、左、上、下四面依次转到可见位置。探测单个 `R/C/F` 操作的四个状态，还能得到每个贴纸位置的 4-cycle，从而恢复全部旋转方向。

本地把 150 个格子展平成一个状态数组，并为 30 个操作预计算置换：

```text
R1..R5, R1'..R5'
C1..C5, C1'..C5'
F1..F5, F1'..F5'
```

这样一次旋转只需对数组做置换，不必再与远端交互。官方 WP 的策略是先把目标字符逐个移动到某一面，并用交换子：

```text
[m1, m2] = m1 · m2 · m1' · m2'
```

尽量减少对已就位字符的破坏。也可以直接对短路径使用 BFS，对较长命令使用带启发式评分的 beam search；评分优先减少缺失字符数、延长可激活前缀，并保留更多横纵交替的候选分支。

### 4. 搜索激活路径

给定一个 5×5 正面和目标字符串，路径约束可直接用 DFS 检查：

1. 第一个字符必须位于第 0 行；
2. 选择第一个字符后，下一字符必须与它同列；
3. 之后依次在“同行”和“同列”之间交替；
4. 光标移动支持 5 格环绕，因此确定坐标后取最短方向即可生成按键序列。

搜索器先枚举少量旋转，使某一面同时包含目标字符并满足上述 DFS；找到后再把该面旋转到正面，回放 `Twist` 序列，切换到 `Activate`，按坐标路径依次选字并执行。

服务端源码在随机打乱前，会在两个随机面上预置 `cd ..;nl flag` 的合法激活路径，并在另外两个面上预置 `ls`。随后才追加 20 次随机旋转。这解释了为什么每局都必然存在目标命令，而不是依赖随机字符恰好排列成功。

### 5. 构造读取命令

先构造 `ls` 可确认当前目录为 `/home/ctf`；再构造 `cd ..;ls`，能看到上一层存在：

```text
../flag
../ctf
```

普通字符池刻意不包含 `/`，所以直接执行 `cat ../flag` 并不稳。利用 shell 的分号先改变目录，就不再需要斜杠。最终选择源码保证可解的命令：

```sh
cd ..;nl flag
```

`cd ..;cat flag` 在部分局面也可能构造出来，但 `cat` 没有预置路径保证；官方设计明确为 `nl` 准备了合法路径，因此应以 `nl` 为标准解。

解码命令输出得到：

```text
SUCTF{Th1s_i5_@_Cub3_bu7_n0t_5ome7hing_u_pl4y}
```

## 方法总结

这题把单表替换、空间状态恢复和受约束命令构造串成了一条链。附件先解决“符文代表什么”，在线界面再解决“怎样把这些字符移到可选路径上”，最后由 shell 语义把“flag 在当前目录之外”的提示转化为 `cd ..;nl flag`。

自动化时应把网络交互压缩到三段：完成 PoW 并采集状态、本地计算全部旋转和激活路径、一次性回放操作。不要把完整求解器的一千多行代码当作题解主体；真正可复用的是四个模型：已知明文恢复替换表、旋转的置换表示、横纵交替路径 DFS，以及在时限内用 BFS / beam search 规划状态。
