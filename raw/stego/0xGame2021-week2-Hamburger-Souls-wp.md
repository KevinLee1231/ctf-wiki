# week2Hamburger Souls

## 题目简述

题面提示“两面包夹 flag”，对应的不是复杂像素隐写，而是把 flag 明文夹在图片文件的数据中。只要在原始字节流中搜索比赛 flag 前缀即可定位。

## 解题过程

用十六进制编辑器打开图片，不必先修改图像内容。在 ASCII 视图中搜索比赛固定前缀 `0xGame{`，可以直接定位到嵌入文件中的完整 flag，并一直读取到右花括号 `}`。

也可以用字符串提取工具完成同样的检查：

```bash
strings -a attachment | grep -o '0xGame{[^}]*}'
```

这里搜索的是文件的原始字节，而不是截图 OCR 或图片元数据；因此即使图片正常显示，也仍可能在像素数据之外包含可打印字符串。

## 方法总结

本题的核心是根据题面联想到“数据被夹在文件中”，然后优先做低成本的明文与文件尾检查。面对媒体附件时，先执行 `file`、`strings`、文件头尾和元数据检查，通常比直接进入复杂隐写分析更高效。
