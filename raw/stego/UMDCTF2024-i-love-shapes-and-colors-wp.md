# i love shapes and colors

## 题目简述

题目只给出 `flag.mp4`，并明确说明它由 “AES-256 + 视频中的几何图形” 生成，密码为
`muaddib`。这里不需要攻击 AES：密钥口令已经给出，真正任务是识别并还原 AES GUI 的专用视频载荷格式，因此归入 Stego。

AES GUI 会先用口令驱动的 256 位 AES 加密文件，再把密文编码成视频帧中的几何形状和颜色。视频不是普通视觉谜题，每一帧的颜色与布局都是编码数据；转码、缩放或改变色彩空间都可能破坏解码。

## 解题过程

### 识别生成工具

题目 README 指向 [AES GUI 官方页面](https://www.matthew-x83.com/aesgui/aesgui.php)。该页面给出的必要机制是：

- 软件运行于 64 位 Windows。
- 任意小文件会先经 AES-256 加密，再表示为视频中的几何图形。
- 接收方使用同一口令从视频无损恢复原文件。
- 视频至少应保持 Full HD 质量和原色彩空间；再次压缩可能引入足以让解码失败的颜色误差。

这些信息足以确定：不应从单帧手工读颜色，也不应先用视频编辑器重新导出。

### 使用原始视频解码

保留仓库中的原始 `flag.mp4`，在隔离的 Windows 环境中运行 AES GUI：

1. 选择从视频恢复文件的解码/解密功能。
2. 输入原始 `flag.mp4`，不要使用经过聊天软件、视频网站或编辑器转码的副本。
3. 口令填写 `muaddib`，保持原始大小写。
4. 选择输出目录并开始恢复。

工具先按帧识别几何形状与颜色，重建 AES 密文，再用口令派生的密钥解密原文件。最终恢复内容给出：

```text
UMDCTF{wtf_is_this_software_im_so_sorry}
```

这里保留官方页面链接是因为它同时提供专用解码器和格式限制；WP 已经写明所需操作与注意事项，不依赖读者再次访问页面才能理解解法。

## 方法总结

- 核心技巧：识别特定软件生成的视频数据载体，使用同一工具和已知口令逆向恢复文件。
- 识别信号：题面同时出现 AES-256、几何形状视频、已知密码和专用工具链接时，重点是格式解码而非密码破解。
- 复用要点：先保存原视频并校验副本，禁止无损性不明的转码；对于按颜色编码的数据视频，肉眼看似几乎不变的压缩伪影也可能让整条恢复链失败。
