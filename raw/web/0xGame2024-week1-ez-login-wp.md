# ez_login

## 题目简述

题目表现为普通用户名、密码登录页，官方解法记录的考点是默认弱口令。登录表单字段分别为 `username` 和 `password`，不需要注入或绕过认证流程。

需要注意：当前源码仓库中的同名目录并不完整一致。`templates/login.html` 是登录页面，但 `main.py` 实际只实现了一个 JSON `merge()` 接口，没有渲染登录页或校验账号；因此不能根据这份 `main.py` 补写线上认证机制，以下仅保留官方题解已经确认的操作。

## 解题过程

在登录表单中提交题目预设的默认凭据：

```text
username: admin
password: admin123
```

登录成功后即可进入显示 flag 的页面。这里的关键不是爆破，而是识别并验证常见默认账号组合。

## 方法总结

- 核心技巧：检查默认凭据 `admin / admin123`。
- 识别信号：没有额外校验、限速或复杂业务逻辑的入门登录题，通常应先测试题目上下文暗示的默认账号。
- 复用要点：WP 只能记录已验证的题目事实；当仓库源码与官方解法不一致时，应明确证据边界，不能用不匹配的源码虚构漏洞链。
