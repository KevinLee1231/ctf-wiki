# week2intval

## 题目简述

题目包含两组必须同时满足的 PHP 类型与正则差异：第一个参数既要不严格等于固定字符串，又要通过以 `^...$` 锚定的正则；第二个参数既要在宽松比较中不等于整数 1024，又要经 `intval()` 后严格等于 1024。

```php
if ($_GET['0xGame'] !== '20201001'
    && preg_match('/^20201001$/', $_GET['0xGame'])) {
    echo 'Good job!<br>';
} else {
    die('Think it over!');
}

$id = intval($_GET['id']);
if ($_GET['id'] != 1024 && $id === 1024) {
    echo 'Congratulations!<br>' . $flag;
}
```

## 解题过程

PCRE 中 `$` 不仅能匹配字符串末尾，也能匹配末尾换行符之前的位置。令第一个值为 `20201001\n`：它与字面字符串不严格相等，但正则仍能匹配。URL 中换行编码为 `%0a`。

第二个值使用 `1024.1`。它与整数 1024 做宽松数值比较时不相等，但：

```php
intval('1024.1') === 1024
```

最终查询串为：

```text
/?0xGame=20201001%0a&id=1024.1
```

请求该路径后，两层条件依次成立并输出 flag。这里依赖题目所用 PHP/PCRE 版本的 `$` 换行语义；若换成绝对结尾锚点 `\z`，第一种绕过将失效。

## 方法总结

- 核心技巧：用末尾换行制造“严格字符串不同但正则仍匹配”，再利用 `intval` 截断小数部分。
- 识别信号：同一输入同时参与严格比较与正则，或同时参与宽松比较与显式整数转换。
- 复用要点：逐个写出每次比较采用的类型和值；防御时使用一致的规范化流程，并用 `\z` 或显式长度检查约束完整字符串。
