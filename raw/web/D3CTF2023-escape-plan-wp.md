# Escape Plan

## 题目简述

题目是 Python Web 沙箱逃逸。服务端对数字、字母和部分字符做过滤，目标是在受限字符集下构造可执行表达式并执行命令。官方解法利用 Unicode 同形字符、`len` 构造数字、字典/list 取字符串片段、base64 编码 payload 等技巧，在没有直接反馈时用带外行为验证命令执行。

## 解题过程

1. 首先，直接访问网页可以看到源代码。

2. 审计源码后可以看出，重点是绕过过滤机制完成沙箱逃逸。

3. 禁用数字：可以用 `len` 这类构造得到 `0` 和 `1`，再逐步组合出需要的数字。

4. 禁用字母：可以用 Unicode 同形字符绕过限制，例如把 `e` 替换为 `ᵉ`。

5. 其他字符：通过 `str(request)` 配合切片，可以取回传入的 payload。

6. 最后，在没有回显的情况下，可以通过带外通信解决验证问题。

从这个角度看，这道题其实比较简单，至少相比本次比赛的其他 Web 题更简单。

不过，上面的做法其实是非预期解。出题时我意外忽略了 import 语句对目标环境的影响，因此所有依赖 `request` 的解法都不在我的预期内。我在比赛第一天才发现这个问题，最初想给出一个修补这个“漏洞”的方案，但考虑到我无法覆盖所有可能情况，而且 Web 题解出数量相对较少，就决定不再增加选手负担，把它当作送给大家的额外入口。

复查提交的 WP 后，我发现大多数解法都基于 `request`。如果感兴趣，可以思考一下：能否不依赖 `request`，而是用另一种方式主动 `import` 想要的包。提交的 WP 中只有少数队伍找到了预期解法，有兴趣的话可以继续挑战。

## 方法总结

- 核心技巧：字符过滤绕过、无数字构造、Unicode 同形字符绕过关键字、通过对象字符串和切片恢复受限字符。
- 识别信号：题目只给一个受限表达式入口，过滤数字/字母/括号等字符，且输出不可见时，应把目标拆成“先构造基础字符和数字，再拼出 `eval`/`__import__`，最后用带外命令验证”。
- 复用要点：不要只依赖 unintended 的 `request` 对象；更稳的思路是从当前命名空间可达对象中恢复 builtin，再用编码后的 payload 降低受限字符直接出现的次数。
