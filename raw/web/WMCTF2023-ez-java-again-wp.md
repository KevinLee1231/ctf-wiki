# ez_java_again

## 题目简述

题目入口是 Java Web 中的图片读取接口，参数 `url1` 会被后端当作 URL 处理。解法利用 `file:///` 协议读取本地文件，并通过百分号编码绕过接口对敏感字符串的过滤，最终读取 `flag`。

## 解题过程

该题关键 payload 为：

`Imagefile?url1=file:///%25%36%36%25%36%63%25%36%31%25%36%37%23java`

其中 `file:///` 让后端按本地文件路径读取内容；后半部分 `%25%36%36%25%36%63%25%36%31%25%36%37` 是双重编码后的 `flag`。第一次解码后得到 `%66%6c%61%67`，再次解码才变成 `flag`，从而绕过只检查明文 `flag` 或只做一次 URL decode 的过滤。末尾的 `#java` 作为 fragment，不参与本地文件路径解析，但可用于满足或干扰后端对扩展名、协议尾部的检查。

## 方法总结

- 核心技巧：`file://` 本地文件读取配合双重 URL 编码绕过敏感路径过滤。
- 识别信号：Java 服务把用户输入当 URL 打开，且过滤逻辑只看明文或单次解码结果时，应测试 `file:///`、`jar:`、fragment 和多轮编码。
- 复用要点：写 WP 时要解释编码层级，否则 payload 看起来只是随机十六进制字符串，不利于复现。
