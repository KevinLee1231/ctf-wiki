# week1view_source

## 题目简述

页面把 flag 放在 HTML 注释中。注释不会显示在正常渲染结果里，但会随 HTML 响应完整发送到客户端，因此查看页面源代码即可读取。

官方附件中的关键内容为：

```html
<!-- < 0xGame{web_sign_in} > -->
```

## 解题过程

打开题目页面后，使用浏览器的“查看网页源代码”（通常为 `Ctrl+U`），或在开发者工具的 Network 面板查看文档响应。也可以直接请求页面并搜索注释：

```bash
curl -s 'http://<HOST>:<PORT>/'
```

在返回的 HTML 中定位 `<!-- ... -->`，即可得到 flag。这里只查看 Elements 面板也可能看到注释，但“页面源代码”或原始响应更直接，且不会受到前端脚本修改 DOM 的影响。

## 方法总结

- 核心技巧：检查客户端实际收到的 HTML 源码，而不是只看渲染页面。
- 识别信号：页面内容简单、提示涉及 source/comment，或可见区域没有输入点。
- 复用要点：HTML 注释、隐藏元素和前端脚本都不具备保密性，敏感信息不应发送到客户端。
