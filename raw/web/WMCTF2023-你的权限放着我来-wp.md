# 你的权限放着我来

## 题目简述

题目是账号权限与密码重置流程缺陷。页面源码泄露了系统预置账号邮箱，其中包含管理员账号 `jom@roomke.com`；密码重置接口 `/api/change` 对 `token` 和邮箱绑定校验不严，导致普通用户可以把重置请求改成管理员邮箱并重放，最终获得管理员权限或 flag。

## 解题过程

程序默认会创建 4 个账号，其中一个是管理员账号（`jom@roomke.com`）。重置管理员账号密码即可拿到 flag。

1. 注册一个账号并登录成功，顺势让参赛者/用户去尝试“忘记密码”这类其它功能；登录成功后在页面上右键查看源码，即可拿到全部邮箱列表，其中包含管理员邮箱。
2. 点击“Forgot Password（忘记密码）”按钮，进入密码找回页并填入目标邮箱。
3. 收到重置邮件中的链接后访问该链接，输入重置密码；抓包记录 `/api/change` 接口请求。
4. 将 `token` 置空，将邮箱改为管理员邮箱，重放抓到的请求包即可拿到 flag。

该利用成立的关键不是邮件本身，而是后端没有强绑定“发起重置的 token、被重置邮箱、当前会话”三者。只要 `/api/change` 接受空 token 或未校验 token 与邮箱的对应关系，就可以把普通用户的重置流程升级成管理员密码重置。

## 方法总结

- 核心技巧：利用密码重置接口中的 token 绑定缺陷重置管理员账号。
- 识别信号：页面泄露用户邮箱、重置接口可抓包修改邮箱、token 可为空或不随目标邮箱变化时，应重点检查越权重置。
- 复用要点：账号找回类题目不要只看邮件链接是否可预测，还要检查最终改密接口是否重新校验 token、邮箱和会话的一致性。
