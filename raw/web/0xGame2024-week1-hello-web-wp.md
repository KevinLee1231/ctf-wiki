# hello_web

## 题目简述

题目提供一个静态首页，并用前端 JavaScript 禁用右键菜单和 `F12`。这只是客户端交互限制，不会阻止浏览器获取 HTML。首页源码的注释给出第一段 flag，并提示继续访问 `/f14g.php`；该 PHP 页面不在正文中显示第二段，而是把它写入 HTTP 响应头 `flag`。

## 解题过程

使用 `Ctrl+U`、在地址前添加 `view-source:`，或者直接查看浏览器已经下载的页面源码。决定解法的注释如下：

```html
<!-- 看看f14g.php -->
<!-- 此乃flag的第一段：0xGame{ee7f2040-1987-4e0a -->
```

随后访问同一站点下的 `/f14g.php`。仓库源码表明该页面通过 `header()` 设置自定义响应头，而正文只返回提示文字：

```php
$flag = '-872d-68589c4ab3d3}';
header('flag: '.'此乃flag的第二段：'.$flag);
echo '你知道如何查看响应包吗？';
```

在浏览器开发者工具的“网络”面板中选择该请求，查看响应头即可得到第二段；Burp Suite、Yakit 或 `curl -i` 也能看到相同内容。

将两段按原顺序拼接，得到：

```text
0xGame{ee7f2040-1987-4e0a-872d-68589c4ab3d3}
```

## 方法总结

- 核心技巧：绕过无安全意义的前端按键限制，检查 HTML 注释和 HTTP 响应头。
- 识别信号：页面刻意禁止右键或开发者工具时，优先确认源码、静态资源、响应头和未直接展示的路径。
- 复用要点：客户端脚本只能影响交互，不能撤回已经发送到浏览器的数据；使用 `view-source:`、开发者工具或 `curl -i` 均可独立验证。
