# 魔理沙的魔法目录

## 题目简述

题面是魔理沙从大图书馆借来很多书，让用户一起研究；只有在线阅读靶机，没有附件源码。页面行为是阅读文章并累计时长，前端网络流中会周期性请求 `record` 接口，上报 `time` 参数，随后由检查接口判断阅读时间是否足够。

## 解题过程

直接访问页面后会发现需要阅读文章 1 小时才能领取 flag。不要等待前端倒计时，打开浏览器网络面板观察请求，可以看到页面周期性调用 `record` 接口，并把 `time` 参数上报给后端；正常情况下每次上报的值是 `10`。

将该请求中的 `time` 改成 `3600` 以上并带上当前登录态重放，后端累计时间会直接达到要求。随后访问检查接口，或者等页面自动触发检查，就能拿到结果。

## 方法总结

遇到阅读、签到、倒计时、进度条类 Web 题，应优先看网络请求而不是等待前端计时。重点确认服务端是否只根据客户端上报参数做累计；如果是，直接篡改时间、进度或次数参数，再触发校验接口。
