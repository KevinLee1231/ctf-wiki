# week1爱ping才会赢

## 题目简述

页面提供 Ping 功能，后端直接把用户提交的 `host` 拼接到系统命令 `ping` 后执行，没有任何转义或白名单校验，因此存在 Shell 命令注入。

## 解题过程

仓库源码中的关键逻辑为：

```php
if (isset($_POST['host'])) {
    system('ping ' . $_POST['host']);
}
```

页面仅给提交按钮添加了 `disabled="disabled"`。这是客户端限制，可以在开发者工具中删除该属性，也可以直接构造 POST 请求；它不会改变后端是否接受 `host` 参数。

先提交正常地址确认功能，例如 `127.0.0.1`。随后利用分号分隔两条 Shell 命令：

```text
host=127.0.0.1;cat /flag
```

后端实际执行的是 `ping 127.0.0.1;cat /flag`，第二条命令读取 flag。仓库内对应结果为：

```text
0xGame{L1nux_cmd_1s_3a5y_t0_you!!!}
```

## 方法总结

决定性缺陷是把不可信输入直接交给 `system()`，而不是按钮是否可点击。修复时应避免调用 Shell；确需执行网络诊断时，应对目标做严格的 IP/域名白名单校验，并使用参数数组形式启动固定程序，不能依靠前端禁用按钮。
