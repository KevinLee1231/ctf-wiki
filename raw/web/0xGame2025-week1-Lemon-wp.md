# Lemon

## 题目简述

页面脚本监听 `contextmenu` 和 `F12`，试图阻止右键菜单与开发者工具，但 flag 实际写在返回 HTML 的注释中。客户端 JavaScript 只能影响页面交互，不能阻止用户读取已经下载到本地的响应源码。

## 解题过程

使用浏览器“查看页面源代码”，或直接请求页面并查看原始响应：

```bash
curl -s 'http://<host>/'
```

在 HTML 末尾可以找到注释：

```html
<!-- 0xGame{Welc0me_t0_0xG@me_2025_Web!!!} -->
```

浏览器快捷键通常为 Windows/Linux 的 `Ctrl+U`，macOS 的 `Command+Option+U`。即使页面拦截 F12，这些方式仍可读取服务端已经返回的内容。

## 方法总结

- 核心技巧：绕过无安全意义的前端交互限制，直接检查原始 HTTP 响应。
- 识别信号：只在 JavaScript 中禁用右键或按键，没有任何服务端鉴权或数据保护。
- 复用要点：浏览器端代码、注释和静态资源都已交付给用户，不能用于保密；优先查看源码、网络响应和静态文件。
