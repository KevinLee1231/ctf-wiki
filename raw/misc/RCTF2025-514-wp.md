### 参考链接补充

`koishi-plugin-pjsk-pptr` 的源码能确认这条链路的两个关键点：

- 插件声明依赖 `puppeteer`，渲染时创建 page/context，把拼出的 `htmlContent` 交给 `page.setContent(htmlContent)`，再调用 `page.screenshot()` 返回图片。
- 角色图片路径使用 `file://` 前缀拼接本地依赖目录，页面资源本身处在本地文件上下文；若用户可控文本被写进 HTML/JS 模板，就可以借同源 `file://` 行为尝试读取本地文件。
- 插件对输入中的 `/` 做了功能性替换，所以 payload 需要用 HTML numeric entity、base64 或拼接方式避开直接斜杠。

因此利用目标不是 Koishi Web 后台，而是 bot 截图渲染链：把 JS 注入到待截图 HTML 中，创建 `iframe src=file:///flag`，再用 CSS 把 iframe 覆盖到 canvas 上，使截图把 flag 内容回显出来。

## 方法总结

- 核心技巧：Puppeteer `file://` origin 下的 HTML/JS 注入。
- 识别信号：插件把用户 text 插入模板并截图渲染，本地文件可被同源 iframe 读取。
- 复用要点：若 `/` 被转义，可用 base64、HTML numeric entity 等方式绕过构造限制。
