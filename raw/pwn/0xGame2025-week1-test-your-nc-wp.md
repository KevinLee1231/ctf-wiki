# test_your_nc

## 题目简述

题目用于确认选手能够使用 Netcat 连接 TCP 服务。连接建立后直接进入一个允许执行基础命令的 Shell，没有额外漏洞利用或认证步骤。

## 解题过程

使用题目实例提供的主机和端口建立连接：

```bash
nc <host> <port>
```

进入 Shell 后先用 `pwd`、`ls -la` 确认当前目录，再读取 flag 文件：

```bash
cat flag
```

如果当前目录没有该文件，再在常见位置检查，而不是盲目执行破坏性命令。

## 方法总结

- 核心技巧：使用 Netcat 与远程交互式 TCP 服务通信。
- 识别信号：题目只提供主机、端口并提示测试 `nc`，连接后直接出现命令提示符。
- 复用要点：先观察服务协议和当前环境，再发送命令；Netcat 只是传输工具，本题没有隐藏的内存破坏步骤。
