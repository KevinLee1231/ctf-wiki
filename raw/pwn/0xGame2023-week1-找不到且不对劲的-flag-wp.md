# 找不到且不对劲的 flag

## 题目简述

程序本身没有内存漏洞：连接后直接执行 `system("/bin/sh")`，考点是 Linux 文件枚举与隐藏目录。当前目录中可见的 flag 是干扰项，提示说明真正的 flag 被“藏”了起来。

## 解题过程

获得 shell 后先确认当前位置并列出包括点文件在内的全部目录项：

```bash
pwd
ls -la
```

Linux 中名称以 `.` 开头的文件或目录默认不会被普通 `ls` 显示。`ls -la` 可发现 `.secret`，进入后读取其中的文件：

```bash
cd .secret
ls -la
cat flag
```

也可以合并为：

```bash
ls -la && cd .secret && cat flag
```

## 方法总结

拿到受限或简易 shell 后，应先做最小环境枚举：检查工作目录、隐藏项、权限和文件类型。不要把第一个看似符合格式的字符串直接当作答案，应结合提示与目录结构验证 flag 的来源。
