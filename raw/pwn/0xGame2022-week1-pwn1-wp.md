# week1pwn1

## 题目简述

本题是 Pwn 方向的网络交互签到题。服务建立 TCP 连接后会直接返回题目信息或 flag，不需要构造内存破坏利用。

## 解题过程

使用题目动态下发的主机和端口建立连接：

```bash
nc HOST PORT
```

读取服务输出并按提示交互即可。原 PDF 只记录“nc 上去即可”，仓库也未保留该题二进制，因此没有可进一步验证的漏洞逻辑；这里不补造栈溢出或 ROP 步骤。

## 方法总结

- 核心方法：直接连接 TCP 服务并读取输出。
- 识别特征：题目没有附件或利用要求，连接后即可获得结果。
- 注意事项：历史比赛 IP 和端口已经失效，不应写死在长期保存的 WP 中；复现时使用平台当前下发的连接信息。
