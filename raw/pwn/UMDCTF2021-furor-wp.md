# Furor

## 题目简述

题目是运行在 Raspberry Pi 风格裸机环境中的 ARMv6 程序。输入函数把数据写入固定栈缓冲区，44 字节后覆盖保存的 PC；内存可执行，程序还提供打印函数和固定地址的 flag。

## 解题过程

对 ARM 反汇编和栈帧布局进行核对，缓冲区起点到保存 PC 的准确距离为：

```text
44 bytes
```

已知关键地址：

```text
flag      = 0x000104f0
printline = 0x000101f0
```

思路是在栈上放一小段 ROP/指令序列，把 `r0` 设置为 flag 地址，再把 `pc` 设置为 `printline`。可利用二进制中的：

```text
pop {r0}
pop {pc}
```

或在可执行栈上注入等价 ARM 指令。payload 结构为：

```text
44 字节填充
覆盖 PC -> 栈上代码或第一个 gadget
0x000104f0
0x000101f0
```

裸机服务可能在接收缓冲区与本地调试环境之间存在固定偏移，公开实例中可按服务提示对栈地址加约 `76` 字节校正；该值应通过返回地址/回显验证，而不是无条件套用。

成功调用打印函数后得到：

```text
UMDCTF-{ARM_L0V3s_R0P}
```

## 方法总结

非 Linux 裸机 pwn 仍遵循控制流利用的基本方法，但不能假设有 libc、系统调用或标准进程内存布局。本题应从 ARM 调用约定出发，让 `r0` 携带字符串指针、`pc` 跳到打印函数；44 字节偏移和 Thumb/ARM 状态都必须以实际指令为准。
