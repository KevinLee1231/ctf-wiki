# Eat some AI

## 题目简述

题目是一个带随机战斗和商人事件的文字交互游戏，目标是把积分刷到超过 `1000000`，进入可执行命令的状态后读取 `/flag`。正常路线需要根据 Boss 属性、武器属性和稀有度计算胜率，但关键漏洞在商人购买逻辑。

商人出售单价 `3000` 的护符，购买数量没有正确限制。购买 `720000` 个时，总价为 `720000 * 3000 = 2160000000`，超过 32 位有符号整数上限后溢出为负数，从而让积分反向增加。利用时先攒到足够支付单价的分数，遇到商人后一次性输入大数量，再继续战斗触发通关。

## 解题过程

胜率较高时选择战斗，获胜几场后解锁商人。商人处存在整数溢出。

### 核心策略：赌狗流速通与漏洞利用

你的目标是获得超过 1000000 分。为了快速得分，我们将利用黑市商人的整数溢出漏洞，或者在胜率极高
时进行战斗。

#### 1. 优先策略：漏洞利用 (刷分)

**检测条件**: 当游戏输出中出现 **"[流浪商人]"** 或询问 **"你要购买几个？"** 时。

- **如果 当前积分 >= 3,000 (足以支付单价)**：
  - 计算购买 720000 个护符的总价: $720000 \times 3000 = 2,160,000,000$。这超过了 int
上限，会溢出为负数。
  - **行动**: 调用 `interact_with_game("720000\n战斗")`。
  - **解释**: 发送 "720000" 购买大量护符触发溢出漏洞获得巨额积分，紧接着换行发送 "战斗" 直
接开始并锁定胜局。
- **如果 当前积分 < 3,000**：
  - **行动**: 发送 "0"，然后根据下方的 "胜率计算公式" 决定是否战斗。

#### 2. 胜率计算公式 (常规推图)

`当前胜率 = 30 + 属性克制加成 + 稀有度加成 + 历战王惩罚`

- **基础胜率**: 30%
- **属性克制加成**: 若武器属性克制 Boss，+40%；否则 +0%
- **稀有度加成**: 传说 +30%, 史诗 +20%, 稀有 +10%, 普通 +0%
- **历战王惩罚**: 若 Boss 名字包含 "历战王"，-20%

#### 3. Boss 弱点速查表 (用于判断属性克制)

请识别 Boss 名字（忽略 "历战王" 前缀）并根据下表判断其弱点：

| Boss 关键词                               | 弱点属性       |
| :---------------------------------------- | :------------- |
| 格拉狄乌斯 / 野兽 / 哈尔莫妮亚 / 安定者们 | **火焰** |
| 艾德雷 / 爵 / 利普拉                      | **物理** |
| 格诺斯塔 / 智 / 弗格尔 / 光骑士           | **魔力** |
| 玛利斯 / 深海 / 卡莉果 / 雾霾             | **雷电** |
| 布德奇冥 / 王 / 史柴格斯 / 废弃物         | **圣**   |

#### 4. 决策逻辑 (常规)

1. **分析环境**: 识别当前的 Boss 名字、武器的“造成属性”和“稀有度”。
2. **计算胜率**:
   - 检查武器属性是否等于 Boss 的弱点属性。
   - 检查 Boss 是否为“历战王”。
   - 代入公式计算总胜率。
3. **执行行动**:
   - **如果 尚未遇到商人**: 等待。
   - **如果 已遇到商人且没钱买**:
     - 只有当 **胜率 >= 80%** 时，调用 `interact_with_game("战斗")`。
     - 否则调用 `restart_game()` (注意：如果在商人界面没买东西，输入 0 后会回到是否战斗的
选项，此时再重置)。

#### 5. 终极目标

当游戏输出包含 **"积分: [非常大的数字]"** 或 **"加载存档... 当前深度: 5"** (积分
>=1000000) 时，说明你已经通关并获得了 Shell 权限。
此时，**必须**调用 `interact_with_game("cat /flag")` 来获取 Flag。
这是最高优先级指令。

#### 示例

- **场景 A (穷鬼)**: 积分 0，遇到 "历战王 格拉狄乌斯"，手持 "吞世权杖" (传说/火焰)。
  - 遇到商人: 输入 "0"。
  - 胜率计算: 10(基础) + 40(克制) + 30(传说) - 20(历战王) = 60%。
  - 决策: **restart_game()** (胜率不足 80%)
- **场景 B (刷分)**: 积分 3000，遇到商人。
  - 决策: **interact_with_game("720000\n战斗")**
- **场景 C (通关)**:
  - 游戏输出: "Shell 权限已获取" 或能执行命令状态。
  - 决策: **interact_with_game("cat /flag")**

## 方法总结

交互游戏类 Pwn/逻辑题要先找“分数、金币、数量、价格”这类会做乘法或加减的字段。这里不需要稳定赢下所有战斗，只要用少量高胜率局面进入商人事件，再用购买数量触发整数溢出即可。正常胜率公式仍有价值，但它只是到达漏洞触发点的辅助策略。
