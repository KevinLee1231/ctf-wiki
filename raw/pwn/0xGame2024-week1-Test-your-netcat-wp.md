# Test your netcat

## 题目简述

这是远程 Pwn 服务连接入门题。程序启动后设置 30 秒超时，输出欢迎信息，随后直接打开 `/flag`、读取全部内容并写到标准输出，不需要构造漏洞利用。

## 解题过程

把题面给出的连接信息写入 `HOST`、`PORT` 环境变量，再建立 TCP 连接：

```bash
: "${HOST:?请设置 HOST 环境变量}"
: "${PORT:?请设置 PORT 环境变量}"
nc "$HOST" "$PORT"
```

连接成功后等待程序输出即可。源码中的核心逻辑为：

```c
int fd = open("/flag", 0);
int len = lseek(fd, 0, 2);
lseek(fd, 0, 0);
char *buf = malloc(len + 1);
read(fd, buf, len);
write(1, buf, len);
```

其中 `write(1, ...)` 的文件描述符 `1` 是标准输出，所以 flag 会直接回到当前网络连接。仓库没有提交容器内 `/flag` 的实际内容，故最终字符串应以当前服务输出为准。

## 方法总结

- 核心技巧：使用 Netcat 连接题目暴露的 TCP 服务并读取输出。
- 识别信号：题面只给主机、端口和连接命令，程序源码没有输入处理或漏洞要求。
- 复用要点：Pwn 远程服务通常把标准输入、标准输出重定向到 socket；先确认能够稳定连接和收发数据，再进入后续利用分析。
