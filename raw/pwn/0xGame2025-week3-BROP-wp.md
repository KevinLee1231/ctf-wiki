# BROP

## 题目简述

题目只开放远程服务，不提供 ELF。核心逻辑如下：

```c
#include <stdio.h>
#include <string.h>
#include <unistd.h>

void init()
{
    setbuf(stdin, 0);
    setbuf(stdout, 0);
    setbuf(stderr, 0);
}

int main()
{
    char s[0x30] = {0};
    init();

    while (strcmp(s, "This_is_the_true_passwd.where_your_get_it?Ur_so_amazing!")) {
        read(0, s, 0x30);
        printf(s);
        memset(s, 0, 0x30);
    }

    puts("Add 0xGame{....} package");
    return 0;
}
```

漏洞位于 `printf(s)`：用户输入被直接当作格式化字符串。题名虽然叫 BROP，但原解法并不是经典 Blind ROP 的崩溃 oracle、stop gadget 和逐 gadget 探测，而是利用格式化字符串在无附件条件下盲读进程内存，重建 ELF 或直接搜索 `.rodata`。

密码常量长 56 字节，而 `read` 最多只写入 `0x30=48` 字节；循环末尾还会清空 `s`。因此直接发送源码中的密码不能可靠退出循环，主线应是从内存取出 `puts` 所使用的真实 flag 字符串。

## 解题过程

### 1. 确定格式化字符串参数序号

先发送带标记的 `%p` 探针，例如逐次测试：

```text
AAAABBBB.%1$p
AAAABBBB.%2$p
AAAABBBB.%3$p
...
```

当输出中出现标记对应的小端整数 `0x4242424241414141` 时，就找到了输入缓冲区或其后附加数据在 `printf` 伪参数列表中的位置。受单次 `0x30` 字节长度限制，可以分多次探测，不必在一个 payload 中枚举所有位置。

### 2. 把 %s 变成任意地址读取

设附加地址对应第 `N` 个格式化参数，把目标地址按八字节对齐放在格式串后，再使用位置参数 `%N$s`。通用布局为：

```text
%N$s<结束标记><对齐填充><目标地址的 p64 小端表示>
```

`printf` 会把第 `N` 个参数解释为 `char *`，从目标地址一直输出到遇见 `\x00`。需要先确认二进制是否关闭 PIE；若基址固定，可从常见 ELF 基址附近探测，并以返回的 `\x7fELF` 魔数确认文件头。若启用 PIE，则先通过 `%p` 泄露代码返回地址，再按静态偏移回推映射基址。

原 WP 保留的[无附件格式化字符串盲读参考](https://xz.aliyun.com/news/12576#toc-3)主要提供了这一方法。即使链接不可用，所需关键信息也就是：找到栈参数序号、用位置参数 `%s` 解引用附加地址、处理空字节并按地址连续重建数据。

### 3. 处理空字节并重建 ELF

`%s` 不是定长读取：遇到 NUL 会立即停止，访问未映射地址还会使当前连接崩溃。因此不能简单地每次固定读取一整页，而应按下面的规则推进：

1. 从当前地址泄露一段字节；
2. 保存收到的字节；
3. 在结果后补回一个 `\x00`；
4. 地址前进 `泄露长度 + 1`；
5. 连接崩溃时重新连接，并根据 ELF 映射边界调整地址。

先重建 ELF 头与程序头，便可确定各个 `PT_LOAD` 段的文件偏移和虚拟地址关系；若只为找 flag，也可以优先扫描只读数据段。保存为 `dump_bin` 后执行：

```bash
file dump_bin
strings -a dump_bin | grep -F '0xGame{'
```

源码中的 `puts("Add 0xGame{....} package")` 表明 flag 作为常量字符串位于二进制只读数据中。远程实例的实际字符串可由上述盲读恢复；公开 PDF 只写了占位符，仓库也没有 Pwn 二进制，因此不能从现有材料中准确补写具体 flag。

## 方法总结

本题的决定性原语是格式化字符串任意读，而不是直接猜密码或传统栈溢出。无附件题的分析顺序是：先用 `%p` 建立栈参数视图，再用位置参数 `%s` 构造任意地址读取，确认 ELF 基址和魔数，最后按 NUL 边界重建映射内容并搜索 `.rodata`。

“盲”只意味着没有本地附件，不意味着可以省略地址校验。必须记录每次泄露的虚拟地址、长度、补回的 NUL 和重连位置，否则拼出的文件会错位。外部参考只用于补充通用方法，WP 正文已经给出完成本题所需的读取原理、异常处理和最终搜索步骤。
