# FMT?

## 题目简述

题目存在格式化字符串漏洞，但保存格式串的缓冲区不在栈上。因此常规做法——在格式串末尾附加 `p64(target)`，再用 `%N$n` 把它当作参数——无法直接使用：`printf` 的伪参数区位于寄存器保存区和栈上，非栈缓冲区中的附加地址不会自然出现在参数列表里。

利用重点是复用栈上已经存在的多级指针关系。先通过格式化字符串改写某个栈指针的低字节，使其指向另一个栈槽；再利用二联与三联指针链，把目标 GOT 地址写进可作为格式化参数读取的栈槽；最后通过该栈槽执行任意地址写，将 GOT 项改成 `system` 或满足约束的 `one_gadget`。

这里“修改栈指针”指的是改写**保存在栈上的指针值**，不是直接修改 CPU 的 `rsp` 寄存器。

## 解题过程

### 1. 建立栈参数与指针链视图

先分批打印位置参数：

```text
%1$p.%2$p.%3$p.%4$p
%5$p.%6$p.%7$p.%8$p
...
```

记录每个参数值，并在调试器中检查它们的解引用关系。需要寻找两类结构：

```text
三联链：参数 A → 栈槽 B → 栈槽 C
二联链：参数 D → 栈槽 E
```

同时泄露栈地址、程序地址和 libc 地址，分别用于确认各栈槽的实际位置、计算 PIE 基址和 libc 基址。栈 ASLR 会改变高位基址，但同一栈映射内相邻槽位通常共享高位，所以改写指针低 1～2 字节即可在附近栈槽之间重定向。

### 2. 用三联链在栈上制造目标指针

`%hhn` 写入当前累计输出字节数的低 8 位，`%hn` 写入低 16 位。假设参数 A 最终能间接改写保存于栈槽 B 的指针，可以先用 `%A$hhn` 或 `%A$hn` 修改 B 中指针的低位，让它转向一个计划作为“伪参数”的栈槽 C。

随后通过链中的下一层写入，把目标地址按小端分成若干半字，逐段写入 C。例如要在 C 中构造 `printf@got`，可依次写入：

```text
printf@got & 0xffff
(printf@got >> 16) & 0xffff
(printf@got >> 32) & 0xffff
```

每次 `%hn` 的填充量按下式计算：已输出字符数为 $printed$、本次目标半字为 $want$ 时，所需增量为

$$
pad = (want - printed) \bmod 2^{16}.
$$

这样做的目的不是立即改 GOT，而是先让某个位置参数真正变成指向 GOT 的可控指针。

### 3. 用二联链完成 GOT 覆写

栈槽 C 已保存目标 GOT 地址后，后续位置参数便可通过 `%N$hn` 对该地址写入。先泄露 libc 地址并计算：

```text
libc_base = leaked_libc_address - known_symbol_offset
system    = libc_base + system_offset
```

再把 `system` 地址按 2 字节拆分，逐段写入 GOT 项。由于 `%hn` 写入值取决于累计输出量，每写一段都要重新计算 `pad`，并按目标半字从小到大安排同一次格式串中的写入顺序，或利用程序循环分多轮完成。

若覆盖 `printf@got` 为 `system`，下一轮把格式串输入改为：

```text
/bin/sh
```

原本的 `printf(user_input)` 就会变为 `system(user_input)`。选择 `one_gadget` 时必须额外验证寄存器和栈约束，不能只算出地址就直接覆盖。

这条路线还要求目标 GOT 可写，即二进制处于 No RELRO 或 Partial RELRO。若 `checksec` 显示 Full RELRO，就必须改写返回地址、函数指针或其他可写控制数据，而不能继续套用 GOT 覆写。

原 WP 的[非栈格式化字符串与栈指针链参考](https://xz.aliyun.com/spa/news/18146)用于补充具体案例。完成本题所需的核心内容已经在正文中展开：枚举位置参数、识别二联/三联链、低位重定向栈上指针、在栈槽构造 GOT 地址、再分段写入 `system`。

公开 PDF 没有给出二进制、具体参数序号、泄露值或最终 payload，仓库也没有对应 Pwn 附件。因此只能准确保留利用机制，不能凭空补造 `%N$hn` 中的 `N`、GOT 地址和比赛 flag。

## 方法总结

非栈格式串的难点不是 `%n` 失效，而是攻击者无法把任意地址直接送进 `printf` 的参数区。解决办法是把现有栈指针当作写入跳板：三联链负责调整“谁指向谁”并在栈上制造目标地址，二联链再把目标值写到该地址。

实际利用必须先画出参数序号、栈槽地址和解引用关系，再逐轮记录累计输出量与每次低位写入。还要先确认 PIE、libc 泄露和 RELRO；若任一保护条件与预期不符，就应切换目标而不是机械复用 GOT 覆写。外链可以帮助理解案例，但不能替代这些针对当前进程栈布局的证据。
