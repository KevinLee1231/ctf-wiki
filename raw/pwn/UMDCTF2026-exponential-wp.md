# exponential

## 题目简述

服务端接收一段最多 100000 字符的 Python 代码，禁止引号、井号、空白和非 ASCII 可打印字符，并要求各字符出现次数排序后恰好为：

$$
1,2,4,8,\ldots
$$

检查通过后直接执行 `exec(c)`。目标是在严苛字符频率约束下构造二阶段执行链。

## 解题过程

Python 启动时导入的 `site` 模块提供内置对象 `quit`，其类型为 `_sitebuiltins.Quitter`。类属性可以被重新赋值，因此可把 `__eq__` 依次改成 `input` 和 `exec`：

```python
quit.__class__.__eq__=input;q=quit==quit;quit.__class__.__eq__=exec;quit==q
```

执行顺序如下：

1. 第一次比较 `quit==quit` 时，`__eq__` 已被替换为 `input`，服务端读取第二阶段代码并保存到 `q`；
2. 再把 `__eq__` 改成 `exec`；
3. `quit==q` 实际调用 `exec(q)`，执行不受第一阶段字符过滤约束的任意 Python。

剩余工作是满足指数频率。官方脚本统计上述有效前缀中每个字符的次数，为每个字符选择不小于当前次数的二次幂目标，再在分号后追加无副作用的赋值和重复字符作为填充。例如通过重复变量名、数字与等号，把所有非零频次补到互不冲突的 $2^k$。

提交第一阶段后，再发送普通 Python 二阶段载荷读取 flag，即可得到：

```text
UMDCTF{exp0N3ntial_Grow7h_1s_too_f4st}
```

## 方法总结

频率检查只约束第一阶段文本，却允许访问已有 Python 对象并修改其类方法。利用比较运算隐式调用 `__eq__`，可以先取得二阶段输入，再交给 `exec`。构造时应先固定最短有效语义，再把频数补到二次幂，而不是反过来从频率表猜代码。
