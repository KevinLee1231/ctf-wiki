# week1Pwn？！

## 题目简述

这是一道用于说明 Pwn 最终交互目标的签到题。远程服务启动后已经提供可交互 Shell，不需要构造溢出、ROP 或 Shellcode；题目附件只用于帮助理解服务形态。

## 解题过程

使用题目当前给出的主机和端口连接服务：

```bash
nc <host> <port>
```

连接后先用 `pwd`、`ls -la` 确认当前目录和文件，再直接读取题目中的 flag 文件：

```bash
cat flag
```

输出即为 flag。历史 WP 没有保留比赛服务地址，而旧地址即使存在也属于已经下线的临时环境，因此不应写回文档。

## 方法总结

本题没有漏洞利用链，重点是理解获得 Shell 后的基本交互：确认当前身份、工作目录和目标文件，再读取 flag。不要因为分类是 Pwn 就机械套用栈溢出模板，先判断服务是否已经直接提供命令执行能力。
