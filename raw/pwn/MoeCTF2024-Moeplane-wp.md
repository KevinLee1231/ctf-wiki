# Moeplane

## 题目简述

程序允许选择发动机并修改推力，但没有正确校验负数组下标。通过 `engine_thrust` 的越界写可以反向覆盖同一结构体开头的 `flight` 字段，把里程改成目标值 `0x1020304050`。

## 解题过程

64 位环境下结构体布局为：

```c
struct airplane {
    long flight;                    // 偏移 0，8 字节
    int altitude;                   // 偏移 8
    int velocity;                   // 偏移 12
    int angle;                      // 偏移 16
    unsigned char engine_thrust[4]; // 偏移 20
};
```

因此 `engine_thrust` 到 `flight` 起始位置相差 20 字节，而不是 12 字节。目标值按小端序存储为：

```text
50 40 30 20 10 00 00 00
```

已有低字节不必全部改动；从 `engine_thrust[-16]` 开始，分别把 `flight` 偏移 4、3、2 的字节写成 `0x10`、`0x20`、`0x30`。程序在访问数组前还会把用户输入减一，所以实际输入的引擎编号依次是 `-15`、`-16`、`-17`。

交互序列为：

```text
1 -15 16
1 -16 32
1 -17 48
```

每行中的第一个 `1` 是修改操作，最后一个十进制数分别对应 `0x10`、`0x20`、`0x30`。若输入先按 32 位无符号整数解析，再被当成有符号下标使用，也可用 `4294967281`、`4294967280`、`4294967279` 表示同一组负数。

## 方法总结

这题把负下标越界写、结构体布局和小端序组合在一起。解题时应先按 ABI 算出每个成员的真实偏移，再把目标整数拆成字节，最后处理程序对索引做的额外 `-1`；只凭反编译器展示的成员顺序容易产生偏移错误。
