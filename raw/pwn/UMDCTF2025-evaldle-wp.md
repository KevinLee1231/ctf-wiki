# UMDCTF 2025 - evaldle

## 题目简述

服务要求每次输入恰好 5 个字符，先按目标单词 `SIGMA` 输出类似 Wordle 的颜色，
随后直接执行：

```python
try:
    exec(guess)
    print("🟩🟩🟩🟩🟩")
except:
    print("🟥🟥🟥🟥🟥")
```

flag 已被读入全局变量 `f`。目标是在 5 字符限制下利用跨轮次状态和异常回显，
逐字符恢复 `f`。决定性机制是 Python jail 逃逸与信息预言机，因此归入 `pwn`。

## 解题过程

`exec` 在模块全局环境中运行，所以前一轮创建的短变量会保留到后一轮。每条语句不足
5 字符时可在末尾补空格。先准备：

```python
I=chr
A=""
```

其中 `A` 保存已经确认的 flag 前缀，`I` 是 `chr` 的短别名。测试候选字符 `c`
时，用三次输入构造候选字符串并与完整 flag 做字典序比较：

```python
Z='c'
B=A+Z
C=B<f
```

若 `B` 是 `f` 的真前缀，或者在当前位置小于正确字符，则 `C` 为 `True`；
反之 `C` 为 `False`。再发送：

```python
C=C-1
I(C)
```

两种分支会变成稳定的异常预言机：

- `B < f`：`C = True - 1 = 0`，`chr(0)` 成功，服务输出绿色结果；
- `B >= f`：`C = False - 1 = -1`，`chr(-1)` 抛出 `ValueError`，服务输出红色结果。

因此可以对可打印字符做二分搜索。确定当前位置后，用以下 5 字符语句把候选字符并入
已知前缀：

```python
A=A+Z
```

从空串开始重复，直到恢复右花括号。Wordle 的前五个颜色不参与利用，只需根据
`exec` 之后额外输出的全绿或全红判断。最终得到：

```text
UMDCTF{that_took_a_lot_more_than_six_guesses}
```

## 方法总结

长度限制并不能隔离执行状态。只要 `exec` 共享命名空间，攻击者就能把复杂表达式
拆成多轮短赋值；而成功/异常的二值差异足以组成比较预言机。修复应移除对用户输入的
`exec`，或在严格解析的白名单语法上实现所需功能，同时避免向客户端暴露可区分的
异常状态。
