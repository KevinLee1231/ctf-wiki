# fmt1

## 题目简述

程序把用户输入直接传给 `printf(buf)`，形成格式化字符串漏洞。栈上已有指针 `ptr=&chg`，而判断要求把 `chg` 从 `0x20ff` 改为 `ans` 的 `0x2023`；只需通过 `%hhn` 覆盖最低一个字节。

## 解题过程

先用 `%p` 或 `%<n>$p` 枚举栈参数，确认 `ptr` 位于第 39 个格式化参数。`%n` 系列会把“当前已输出字符数”写入参数指向的地址，`%hhn` 只写最低 1 字节。

`0x23` 的十进制是 35，因此载荷为：

```text
%35c%39$hhn
```

`%35c` 使累计输出字符数变为 35，`%39$hhn` 将 `0x23` 写到 `chg` 的低字节；原高字节 `0x20` 保持不变，最终 `chg=0x2023`，与 `ans` 相等并进入 `system("/bin/sh")`。

本题特定编译布局下还存在非预期解 `%39$lln`：在尚未输出字符时向 `chg` 起始位置写入 8 字节零，恰好同时清零相邻的 `chg` 和 `ans`，使二者相等。但它依赖局部变量相邻顺序，不如单字节写法稳定。

```python
from pwn import *

io = process("../dist/fmt1")
io.sendafter(b"content: ", b"%35c%39$hhn")
io.interactive()
```

## 方法总结

格式化字符串写入要先确认目标指针的参数序号，再根据目标值选择 `%hhn`、`%hn` 或 `%n` 并计算累计输出量。短写入能减少对相邻内存的破坏；依赖偶然栈布局的宽写入只能作为特定版本的补充解法。
