# ipv8

## 题目简述

程序模拟 IPv4 路由配置。源地址使用无长度限制的 `scanf("%s", src_addr)` 读入，而源地址缓冲区只有 `0x30` 字节，存在栈溢出。程序随后还会验证地址格式，并在错误时退出，因此载荷既要覆盖返回地址，也要让检查流程正常返回。

二进制未开启 PIE，`win` 地址固定为 `0x402f45`。

## 解题过程

反汇编可确定：

- `src_addr` 位于 `rbp-0x60`；
- 保存的返回地址相对 `src_addr` 偏移为 `0x68`；
- `dest_addr` 位于 `rbp-0xc0`；
- `dest_asn` 紧接在其后，位于 `rbp-0x90`。

源地址先放一个恰好含三个点的合法前缀，再填充到返回地址：

```python
from pwn import *

payload = b"1.1.1.1"
payload += b"A" * (0x68 - len(payload))
payload += p64(0x402f45)
```

`p64` 中的零字节会使后续字符串验证在覆盖返回地址前停止观察，因此前三个点仍满足地址检查。

还有一个退出条件需要处理。目的地址由 `scanf("%48s", dest_addr)` 读取；输入恰好 48 个非空白字符时，`scanf` 自动追加的终止零会写到紧邻的 `dest_asn` 首字节，把它清零。目的地址本身仍必须恰好包含三个点，可使用 `b"1.1.1.1" + b"A" * 41`。这样地址检查通过，`check_rine` 又不会命中字符串 `"0.0.0.0"` 的退出分支，函数可以正常返回到被覆盖的 `win`。

交互顺序为：

```text
源 ASN：任意值
源地址：上面的溢出载荷
目的 ASN：任意值
目的地址：`1.1.1.1` 后接 41 个 `A`，合计 48 字节
```

进入 `win` 后读取：

```text
UMDCTF{why_was_ipv9_afraid_of_ipv7?ipv789}
```

## 方法总结

本题不只是计算返回地址偏移。源地址载荷要利用 NUL 让字符串验证看见一个合法前缀；目的地址还要借助 `scanf` 的自动终止符清零相邻字段，避免控制流提前退出。两个输入原语配合后，ret2win 才能真正落地。
