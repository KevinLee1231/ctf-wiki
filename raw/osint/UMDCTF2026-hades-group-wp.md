# hades-group

## 题目简述

题目提供一个虚构犯罪团伙的 Telegram JSON 导出，要求识别只以匿名频道身份发言的群主，并沿题目提供的、仅含虚构种子数据的 OSINT 查询链取得最终文档记录号。

导出中混有大量普通成员和诱饵。群主没有个人账号发言，因此不能从参与者列表直接选人；真正的入口是匿名频道消息中留下的贴纸包痕迹。

## 解题过程

先按 `from_id` 过滤所有以 `channel` 开头的消息，再检查其中的附件、贴纸和链接。唯一具有稳定归属线索的对象是消息 56 使用的贴纸包：

```text
styx_reaction_pack
```

随后严格使用题目列出的查询工具，并在每一步用另一数据源交叉验证：

1. 将贴纸包名交给贴纸包情报工具，取得该虚构贴纸包的创建者用户名；
2. 用用户名历史工具查询该用户名的旧名和变更记录，排除同名诱饵；
3. 将当前名、旧名分别送入两套泄漏库，核对 Telegram UID 或电话号码；
4. 用 Telegram 缓存资料库查询用户名与电话号码的关联，确认号码并非仅由单一泄漏源给出；
5. 根据号码所属国家选择对应的区域文档库，取得姓名与国家；
6. 最后把“姓名 + 国家”交给通用文档库，得到题目要求的 Document Record ID。

仓库保存了导出和最终 flag，但没有保存这些临时 bot 的逐步响应，因此离线材料不足以诚实复原每个中间姓名、号码和 UID。可以确认的证据链起点是 `styx_reaction_pack`，终点记录号为：

```text
REC-9305174
```

按题目格式提交：

```text
UMDCTF{REC-9305174}
```

## 方法总结

这道题的重点是从匿名频道内容寻找可归属的外部对象，而不是对所有群成员做无差别查询。贴纸包名连接到创建者身份，用户名历史、两套泄漏库和缓存资料用于交叉验证，区域文档库再把号码转换为实名，最终才查询记录号。题目明确限定所有身份均为虚构种子，实际 OSINT 中也应遵守授权范围并避免查询真实个人。
