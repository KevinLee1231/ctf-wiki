# MemeCTF

## 题目简述

题目只给出一段 SHA-256 和与表情包有关的提示。哈希本身不可逆，正确方向是把它作为精确搜索词，在公开代码历史中寻找同一字符串。

## 解题过程

用完整哈希加引号搜索，可以定位到 UMD Cybersecurity Club 的公开 GitHub 仓库 `UMD-CSEC/hmm`。继续查看提交历史，而不只查看当前分支，可以找到提交信息 “Added flag file”；该历史提交新增了 `lol.jpg`。

打开该历史版本的图片即可读到：

```text
UMDCTF-{meme_ch4llenges_ftw}
```

关键情报已经完整写入正文，因此不依赖仍可能变化的搜索结果页或仓库页面也能复现调查路径。

## 方法总结

OSINT 中的哈希常被当作唯一标识符，而不是要求碰撞或暴力破解。搜索后还要检查版本历史，因为目标文件可能已从当前分支删除；提交信息和旧树对象往往比仓库首页更有价值。
